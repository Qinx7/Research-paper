# 学术搜索权威筛选增强总纲

## 1. 目标

增强当前学术搜索中的“权威筛选”能力，重点不是盲目增加标签数量，而是把现有标签的**核验语义、展示分层、识别准确性**做清楚，避免把“待核验”误展示成“已核验”。

## 2. 需求理解

当前用户对权威筛选的核心诉求是：

1. `IEEE / ACM / 北大核心` 这类当前已有一定判断基础的标签，要更准、更稳
2. `EI / JCR / 中科院分区` 这类目前不能稳定完成真实核验的标签，必须和“已核验”明确区分
3. 搜索结果卡片中，权威标签默认可见，但待核验标签必须显著标注为 `待核验`
4. 每篇文献卡片优先使用“简洁徽标型”标签，不增加过长解释，避免压垮结果密度

## 3. 已确认产品决策

### 3.1 标签分层

搜索结果中的权威标签分为两层：

1. `已核验`
   - 当前可以较稳定判断的权威标签
   - 例如：`IEEE`、`ACM`、`北大核心`
2. `待核验`
   - 当前仅能做候选推断，不能宣称已经核验成功
   - 例如：`EI`、`JCR`、`中科院分区`

### 3.2 展示策略

每篇文献卡片：

1. 默认直接显示标签
2. 标签样式使用“简洁徽标型”
3. 对待核验标签使用明确前缀或样式区分，例如：
   - `待核验 EI`
   - `待核验 JCR`
   - `待核验 中科院分区`

### 3.3 设计原则

1. 不把“候选命中”包装成“已认证”
2. 不为了前端好看而牺牲语义准确性
3. 前端必须让用户一眼看出哪类标签已经确认，哪类只是待核验

## 4. 范围边界

本轮优先做：

1. 标签语义分层清晰化
2. 已核验标签识别策略稳定化
3. 待核验标签前端展示规范化
4. 搜索摘要与结果卡片的标签一致性

本轮不做：

1. 接入新的付费目录或官方授权库
2. 承诺 `EI / JCR / 中科院分区` 已实现官方目录级别核验
3. 新增大量复杂解释文案塞进每篇结果卡片

## 5. 技术现实约束

1. 当前后端已有 `authority_filter_service`，但对不同标签的语义层级需要进一步明确
2. 某些目录标签没有可稳定、公开、低成本的实时核验来源
3. 因此系统必须支持“可信候选”与“已确认标签”的并存，而不是二元强判

## 6. 实施原则

1. 先收敛语义，再收敛样式
2. 先改现有权威标签的判定与输出结构，不扩散到无关搜索逻辑
3. 前后端统一一套字段语义，避免前端二次猜测
4. 保持当前卡片信息密度，优先使用短徽标表达

## 7. 阶段拆分

### 阶段1：标签语义与输出结构收敛

目标：

1. 明确“已核验 / 待核验”的后端输出语义
2. 对 `IEEE / ACM / 北大核心` 与 `EI / JCR / 中科院分区` 做分层
3. 搜索摘要、单篇结果、来源说明使用统一结构

### 阶段2：已核验标签判定增强

目标：

1. 增强 `IEEE / ACM / 北大核心` 的命中规则
2. 减少误判与漏判
3. 尽量让当前可判定标签更稳定

### 阶段3：前端结果卡片样式增强

目标：

1. 在搜索结果卡片中默认显示权威标签
2. 简洁徽标呈现
3. 待核验标签视觉上和已核验标签明显不同

### 阶段4：摘要与诊断一致性校验

目标：

1. 搜索摘要中的 `authority_summary` 与单篇卡片展示一致
2. 待核验数量、已核验数量不自相矛盾
3. 避免前端出现“摘要说待核验，卡片却像已确认”的问题

## 8. 风险

1. 若后端字段语义不统一，前端会继续出现误导展示
2. 若前端只靠颜色区分，用户可能仍然看不出“待核验”
3. 若判定规则改动过猛，可能影响当前搜索结果排序或标签稳定性

## 9. 成功标准

1. 搜索结果卡片能明显区分 `已核验` 与 `待核验`
2. `IEEE / ACM / 北大核心` 的识别结果更稳定
3. `EI / JCR / 中科院分区` 不再被误解为已确认标签
4. 搜索摘要、卡片、来源诊断三处标签语义一致
