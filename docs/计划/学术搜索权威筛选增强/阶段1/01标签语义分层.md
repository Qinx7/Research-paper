# 阶段1 01 标签语义分层

## 目标

先把权威标签的语义层级和输出结构定清楚，确保后续前后端都建立在统一定义上。

## 当前问题

目前最大问题不是“标签不够多”，而是：

1. 某些标签已经有真实判定能力
2. 某些标签仍然只是候选推断
3. 但前端如果展示不清，就会让用户误以为它们是同等级确认

## 本阶段要解决的事

1. 后端输出中，明确区分：
   - `authority_tags`
   - `pending_authority_tags`
2. `verified_level`、`authority_level` 等字段语义进一步收敛
3. `authority_summary` 中分别统计：
   - 已核验标签数量
   - 待核验标签数量
4. 单篇论文对象和摘要对象使用统一语义

## 标签分组建议

### 已核验标签

优先放入：

1. `IEEE`
2. `ACM`
3. `北大核心`

要求：

1. 只有满足当前规则时才进入 `authority_tags`
2. 卡片直接显示为普通权威徽标

### 待核验标签

优先放入：

1. `EI`
2. `JCR`
3. `中科院分区`

要求：

1. 进入 `pending_authority_tags`
2. 卡片默认显示，但必须带 `待核验` 语义

## 输出结构要求

单篇结果建议至少保持：

1. `authority_tags: string[]`
2. `pending_authority_tags: string[]`
3. `authority_reasons: string[]`
4. `verified_level`
5. `authority_level`

摘要建议至少保持：

1. `verified_counts`
2. `pending_counts`
3. `has_verified`
4. `has_pending`
5. `overview`

## 验证点

1. 同一篇文献不会把同一个标签同时放进已核验和待核验
2. 摘要中的数量与卡片标签总数对得上
3. 前端能够不靠猜测直接渲染
