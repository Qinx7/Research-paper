# PRD v2 实施路线图

## 1. 目标

把 `docs/产品/PRD_v2_产品收敛版.md` 变成可连续执行的实施路线，而不是停留在产品描述层。

## 2. 实施原则

1. 先做 P0 主链路
2. 先稳定状态一致性，再扩页面
3. 先兼容旧数据，再引入新字段
4. 不把中长期能力混进当前主链路

## 3. 路线总览

### 阶段1：搜索与三步分析稳定化

目标：

1. 搜索记录恢复稳定
2. 搜索结果态与三步分析不重复生成
3. 来源诊断、缓存恢复、删除体验完整

### 阶段2：项目知识底座闭环

目标：

1. 项目知识工作台可见
2. 成果入库可操作
3. 证据卡片、草稿、成果状态可汇总

### 阶段3：写作状态模型收敛

目标：

1. `/writing` 与 `PaperWorkflow` 共享状态规则
2. 章节引用、真实性标记、保存状态一致
3. 当前章节知识依据可追溯

### 阶段4：开题主流程产品化

目标：

1. 需求理解 → 文献分析 → 研究方向 连续入口
2. 研究方向 → 项目设计 承接
3. 开题报告 / 开题 PPT 入口统一

### 阶段5：交付与答辩链路

目标：

1. 草稿导出
2. 答辩大纲
3. 交付状态工作台

## 4. 当前执行建议

结合当前代码实际，建议执行顺序为：

1. 阶段1 已基本进入收尾
2. 阶段2 与阶段3 正在进行
3. 下一轮优先推进阶段3收口，再进入阶段4

## 5. 成功标准

1. 用户再次进入已有搜索结果时不重复生成
2. 项目页能展示并操作知识沉淀
3. 写作页能稳定使用并追溯依据
4. 开题主流程有明确入口，不靠用户自己拼页面
