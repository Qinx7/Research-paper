# 论文型 PDF 结构化结果前端展示总纲

## 1. 目标

把后端已经产出的论文型 PDF 轻量结构化结果，转换成用户可见的前端能力。优先展示：

1. 抽取标题
2. 抽取摘要
3. 参考文献区是否识别到
4. 抽取置信度

## 2. 现状

当前后端对 `scholarly_pdf` 已经能够输出：

1. `document_kind`
2. `structured_fields`
3. `structured_content`
   - `title`
   - `abstract`
   - `references_text`
4. `structured_confidence`

但这些信息还没有在前端成果管理区或项目知识工作台中展示，用户感知不到这项能力。

## 3. 范围边界

本轮做：

1. 在成果管理区展示论文型 PDF 的结构化结果
2. 明确标识这是“自动抽取结果”
3. 显示标题、摘要预览、参考文献区状态和置信度

本轮不做：

1. 参考文献逐条展开
2. 结构化内容人工编辑
3. 文本高亮到 PDF 原页
4. 新建独立论文阅读页

## 4. 推荐入口

优先放在：

1. `OutcomeManager`

原因：

1. 用户上传成果后最先会在这里看状态
2. 结构化结果本身属于“解析结果的一部分”
3. 不需要新增页面就能形成可见价值

## 5. 推荐展示方式

每个成果卡片在“已入库”且 `document_kind === scholarly_pdf` 时，增加一个轻量结构化结果区：

1. 标题
2. 摘要前几行
3. 参考文献区：已识别 / 未识别
4. 置信度：`title / abstract / references`

并加一句说明：

> 该结果来自自动规则抽取，后续如需更高精度可升级学术论文结构解析服务。

## 6. 风险

1. 抽取结果可能不稳定，若直接当正式元数据展示会误导用户
2. 卡片信息太多会压缩成果管理区可读性

## 7. 控制策略

1. 只展示论文型 PDF
2. 默认展示摘要预览，不展示完整 `references_text`
3. 显式标注“自动抽取”
4. 置信度用简洁短标签呈现

## 8. 成功标准

1. 用户能在成果卡片中看到论文型 PDF 的结构化结果
2. 结果展示不会挤坏当前成果管理区
3. 抽取信息以辅助信息呈现，不会被误解为人工校对结果
