# 阶段2 01：自动推进与项目沉淀

## 目标

在阶段 1 的三步结果展示基础上，补齐自动推进和保存能力，让用户可以把选题研究结果沉淀到项目中。

## 实现内容

### 1. 自动推进

默认流程：

```text
用户提交需求
↓
需求理解开始
↓
文献检索开始
↓
文献返回后自动分析文献
↓
文献分析完成后自动生成研究方向
```

用户不需要逐步点击“下一步”。

### 2. 单块重试

每块卡片提供独立重试：

- 重新理解需求；
- 重新分析文献；
- 重新生成方向。

重试下游卡片时，应保留上游结果，避免用户重复等待。

### 3. 保存方向到项目

用户点击某个方向的“保存到项目”：

- 未登录：跳转登录；
- 已登录且已选择项目：把方向保存到当前项目；
- 已登录但无项目：自动创建一个以当前研究需求命名的项目，再保存方向。

保存成功后提示：

```text
已保存到项目，可在研究方向页继续查看。
```

### 4. 跳转研究方向页

方向保存后提供入口：

```text
进入研究方向页
```

跳转到 `/research`，由现有研究方向页面读取项目下的方向数据。

## 验证方式

- 搜索后可自动完成三步。
- 可以只重试“研究方向”而不重新检索文献。
- 未登录保存方向会引导登录。
- 已登录保存方向后，`/research` 能看到对应方向。
- `npm.cmd run build` 通过。
