# 选题研究三步入口总纲

## 需求理解

当前首页已经具备类似学术搜索产品的极简检索入口和检索结果态，但“开题前半段”的能力还没有形成连续体验。用户在早期并不是马上要生成开题报告或 PPT，而是先想知道：

- 系统是否理解自己的研究想法；
- 当前文献能说明什么研究现状和不足；
- 这个想法可以转化成哪些可做的研究方向。

因此，本方案只聚焦三步：

```text
需求理解 → 文献分析 → 研究方向
```

本轮不接入项目设计、开题报告、开题 PPT，也不恢复旧的 `pipeline` 页面。

## 设计目标

- 让首页搜索后直接变成“选题研究”结果区，而不是单纯文献列表。
- 保留右侧 References 文献来源面板，保证所有分析和方向都有可追溯依据。
- 用三块清晰结果卡片承接现有后端能力：需求理解、文献分析、研究方向生成。
- 让用户可以在确认方向后保存到项目，再进入 `/research` 做更完整的方向仪表盘。

## 推荐加载位置

入口加载在首页 `HomeSearchLanding` 的检索结果态主内容区。

当前首页结构：

```text
左侧窄导航
中间主内容区
右侧 References 面板
```

建议调整为：

```text
左侧窄导航
中间：选题研究三步结果
右侧：References 文献依据
```

其中中间主内容区按顺序展示：

1. 需求理解卡；
2. 文献分析卡；
3. 研究方向卡。

## 关键交互原则

### 1. 不做重型工作流

界面不使用“步骤条 + 下一步”式 pipeline。用户输入需求后，系统自动推进三块结果：

```text
先显示需求理解
文献检索完成后显示文献分析
文献分析完成后显示研究方向
```

用户可以重新生成某一块，但默认不需要手动一步步点击。

### 2. 主区域负责结论，右侧负责证据

- 主区域展示理解、分析和方向。
- 右侧 References 保持文献列表、来源状态、保存文献到项目。
- 研究方向卡中只展示关键摘要，不堆满文献详情。

### 3. 只做选题，不做完整开题

本入口只回答“这个选题怎么形成”。不展示：

- 项目设计；
- 开题报告；
- 开题 PPT；
- 答辩 PPT；
- 论文写作。

这些能力后续仍放在项目详情页或论文工作区中。

## 三块卡片设计

### 需求理解卡

数据来源：`analyzeRequirement(query)`

展示内容：

- 研究领域；
- 核心技术；
- 应用场景；
- 研究对象；
- 可能研究方法；
- 适合成果形式；
- 初步研究建议。

用户操作：

- 重新理解需求；
- 使用当前理解继续分析。

### 文献分析卡

数据来源：

1. `searchLiterature(...)` 返回文献；
2. `analyzeLiterature({ papers, requirement })` 生成分析。

展示内容：

- 国内研究现状；
- 国外研究现状；
- 研究热点；
- 研究不足；
- 可切入点；
- 文献依据数量和来源概览。

用户操作：

- 重新分析文献；
- 打开或关闭右侧 References；
- 保存关键文献到项目。

### 研究方向卡

数据来源：`generateDirections({ literatureAnalysis, requirement, projectId })`

展示内容：

- 3-5 个候选研究方向；
- 每个方向显示题目、研究问题、研究方法、创新点、可行性评分、推荐指数；
- 标注“依据来自当前检索文献分析”。

用户操作：

- 保存方向到项目；
- 进入 `/research` 查看完整方向仪表盘；
- 返回修改需求重新生成。

## 数据流

```text
用户输入研究想法
↓
analyzeRequirement
↓
searchLiterature
↓
analyzeLiterature
↓
generateDirections
↓
保存方向 / 进入研究方向页
```

若用户未登录：

- 可以完成搜索和三步预览；
- 保存文献、保存方向、进入项目沉淀时引导登录。

若用户已登录但未选择项目：

- 保存方向时自动创建一个以需求命名的项目；
- 文献可保存到该项目；
- 方向生成时传入 `projectId` 进行持久化。

## 异常与空状态

- 文献检索为空：文献分析卡显示“暂无足够文献依据”，研究方向卡不生成或只给低置信度建议。
- 文献来源失败：右侧 References 显示各源状态，主区域提示当前分析依据有限。
- 文献分析失败：保留需求理解和文献列表，允许重试分析。
- 研究方向生成失败：保留前两块结果，允许重试方向生成。
- 未登录保存：跳转登录，登录后再保存。

## 不做内容

本阶段明确不做：

- 完整开题工作流；
- 项目设计生成入口；
- 开题报告生成入口；
- 开题 PPT 入口；
- 新增数据库表；
- 新增依赖；
- 大范围重写 `/research` 页面。

## 验收标准

- 首页检索后，中间主区域能看到需求理解、文献分析、研究方向三块内容。
- 右侧 References 仍能展示文献来源、来源状态和保存文献能力。
- 研究方向可以保存到项目，并能在 `/research` 中查看。
- 检索失败、分析失败、未登录保存等状态有明确提示。
- 前端构建通过。

## 建议实施顺序

1. 阶段 1：新增三步结果面板的最小 UI，只串联现有 API。
2. 阶段 2：补自动推进、重试、保存方向到项目。
3. 阶段 3：补状态细节、来源依据提示和体验打磨。
