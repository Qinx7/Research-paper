# 阶段1 01：三步结果面板最小实现

## 目标

在首页检索结果态中新增“选题研究”三步结果面板，先完成最小可用闭环：

```text
需求理解 → 文献分析 → 研究方向
```

本步骤只做最小实现，不改后端接口，不新增依赖，不接入项目设计、开题报告或 PPT。

## 修改范围

- `frontend/src/components/HomeSearchLanding.tsx`
- 必要时新增轻量展示组件，例如 `TopicResearchPanel.tsx`
- 复用现有 `frontend/src/lib/api.ts` 中的：
  - `analyzeRequirement`
  - `searchLiterature`
  - `analyzeLiterature`
  - `generateDirections`

## 实现内容

### 1. 结果态主区域替换

检索后，中间主内容区不再只展示普通搜索回答，而是展示三块卡片：

- 需求理解；
- 文献分析；
- 研究方向。

右侧 References 面板保持不变。

### 2. 需求理解卡

用户提交搜索后，优先调用 `analyzeRequirement(activeQuery)`。

展示字段：

- 研究领域；
- 关键词；
- 研究对象；
- 研究方法；
- 初步建议。

若接口失败，卡片显示“需求理解失败，可重试”，但不阻断文献检索。

### 3. 文献分析卡

文献检索成功且存在 `papers` 后，调用：

```ts
analyzeLiterature({ papers, requirement: activeQuery })
```

展示字段：

- 国内研究现状；
- 国外研究现状；
- 研究热点；
- 研究空白；
- 可切入点。

若文献为空，不调用分析接口，显示“暂无足够文献依据”。

### 4. 研究方向卡

文献分析完成后，调用：

```ts
generateDirections({ literatureAnalysis, requirement: activeQuery, projectId })
```

展示 3-5 个方向卡片：

- 题目；
- 研究问题；
- 方法；
- 创新点；
- 可行性和推荐评分。

第一阶段只展示结果，保存方向的交互放到阶段 2。

## 状态设计

- `requirementLoading`
- `literatureAnalysisLoading`
- `directionsLoading`
- `requirementError`
- `literatureAnalysisError`
- `directionsError`

每块卡片独立 loading / error，避免一个环节失败导致整页空白。

## 验证方式

- 首页输入研究需求后，能看到三块结果区域。
- 文献为空时不生成文献分析和方向，并显示明确提示。
- 任一接口失败时，其他已完成卡片不消失。
- `npm.cmd run build` 通过。
