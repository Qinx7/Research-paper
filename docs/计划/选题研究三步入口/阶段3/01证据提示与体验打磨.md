# 阶段3 01：证据提示与体验打磨

## 目标

在三步入口可用后，补充证据提示、空状态和视觉细节，让用户明确知道每个结论来自哪些文献，以及当前结果的可靠程度。

## 实现内容

### 1. 证据强度提示

文献分析卡和研究方向卡增加依据提示：

- 本次分析使用文献数量；
- 中文文献数量；
- 英文文献数量；
- 主要来源数据库；
- 是否存在来源异常。

示例：

```text
本次分析基于 12 篇文献，其中中文 4 篇、英文 8 篇。知网当前请求失败，中文依据可能不足。
```

### 2. 低依据警告

当文献数量不足或来源过于单一时，给出提示：

- 少于 3 篇文献：不建议直接生成研究方向；
- 没有中文文献：提示中文研究现状不足；
- 没有英文文献：提示国际研究现状不足；
- 只有单一来源：提示需要补充交叉来源。

### 3. References 联动

研究方向卡可以展示简短“依据来源”摘要：

- 方向相关文献标题；
- 来源数据库；
- 年份；
- 引用量。

第一版不做复杂高亮联动，只提供清晰来源摘要。

### 4. 视觉细节

保持首页当前 Consensus 风格，不引入新的重视觉体系：

- 卡片分层清晰；
- loading 使用轻量骨架或状态文本；
- 错误提示温和但明确；
- 右侧 References 不遮挡主内容。

## 验证方式

- 文献来源不足时有明确提示。
- 研究方向卡能看到依据摘要。
- References 打开/关闭不会破坏三步面板布局。
- `npm.cmd run build` 通过。
