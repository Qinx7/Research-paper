# 聊天页配色改版总纲

## 需求理解

本轮仅调整 `AI 学术对话页` 的视觉配色与界面气质，使其接近用户提供的学术搜索网站参考设计。

范围限制：

- 仅修改聊天页及其直接依赖组件的颜色、边框、背景、阴影、标签、状态样式。
- 不改首页。
- 不改后端接口、数据结构、交互流程与页面整体骨架。
- 颜色参考“清透科技感”，但不照抄原设计色值。

## 已确认方向

- 页面范围：只改聊天页
- 视觉方向：A 轻雾研究台
- 目标气质：清透科技感、冷白底、浅青蓝点缀、专业检索台风格

## 关键假设

- 当前三栏研究台布局继续保留，不再做结构级重排。
- 主要改动集中在 `frontend/src/app/chat/page.tsx` 与 `frontend/src/components/chat/` 下的聊天相关组件。
- 当前页面功能已可用，本轮优先做视觉统一，不顺手扩展新功能。

## 建议改动点

1. 全局底色从当前偏 `slate` 灰改为更亮的冷白雾面底。
2. 主强调色统一为青蓝系，覆盖按钮、选中态、状态点、标签、链接。
3. 聊天正文区、历史侧栏、证据栏统一层次语言：
   - 面板底色更轻
   - 边框更细更浅
   - 阴影更柔和
4. 用户消息气泡改为更清透的蓝青色，不使用过重高饱和块面。
5. 助手回答卡、证据卡、来源状态卡统一为“检索结果卡片”风格。
6. 输入控制台改成更接近专业搜索产品的冷白控制条观感。

## 预期结果

- 聊天页整体更接近参考稿的清透科技感。
- 页面仍保持高效率学术检索工作台的专业气质。
- 现有功能与交互不受影响。

## 验证方式

- 执行 `npm.cmd run build`
- 目测聊天页：
  - 历史侧栏、对话区、证据栏配色统一
  - 输入区与检索控制条视觉连贯
  - 选中态、按钮态、状态点颜色一致
  - 文本对比度可读
