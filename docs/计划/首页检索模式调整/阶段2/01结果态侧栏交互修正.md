# 阶段2 01：结果态侧栏交互修正

## 反馈理解

用户补充了两张 Consensus 截图：

- 首页初始态应该是极简搜索页，左侧窄导航，中央品牌标题和搜索框；不应该一开始就出现右侧 References。
- 搜索之后才进入结果态，右侧 References 面板才出现。
- References 面板需要可以打开和关闭。
- 右侧栏中的文献点击后可以展开内容。
- 当前右侧栏固定死且不能独立滑动，导致整个页面变长，需要改为面板内部滚动。

## 最小修正方案

1. 首页初始态恢复为接近截图 1 的布局：
   - 左侧窄边栏。
   - 中央居中标题、搜索框和快捷按钮。
   - 不显示右侧 References。
2. 搜索后进入结果态：
   - 左侧仍是窄边栏。
   - 中央显示问题、检索路径和结果概览。
   - 右侧 References 默认打开，但提供关闭按钮。
3. References 交互：
   - 顶部提供关闭按钮。
   - 中央结果区提供“打开 References”按钮。
   - 面板设置固定高度 `h-screen`，内部列表使用 `overflow-y-auto`，不再撑高整页。
   - 文献卡片点击展开/收起，展开后显示摘要、作者、来源、引用数和链接。
4. 不改后端、不改检索策略、不删除 `pipeline` 页面。

## 验证标准

- 初始首页没有右侧 References 面板。
- 提交搜索后才出现 References。
- 点击关闭按钮后右侧栏消失，主内容自动占满空间。
- 点击“打开 References”后右侧栏恢复。
- References 内部可以独立滚动。
- 点击任意文献卡片可以展开/收起详情。
- 前端构建通过。
