# 首页检索模式调整总纲

## 需求理解

当前首页输入检索词后会直接跳转到聊天页或文献搜索界面。用户希望首页搜索模式参考 Consensus 的学术搜索体验：输入问题后先在首页内部展示研究结果、检索依据和 References，不要一输入就离开首页。

`pipeline` 页面可以暂时不用，但本轮不删除页面，避免破坏历史入口或潜在引用。

## 关键假设

- 本轮只调整首页搜索入口和首页内结果展示，不改后端检索体系。
- 首页提交检索时直接调用已有 `searchLiterature` API，先展示可核验文献结果，而不是进入聊天页。
- 聊天页继续作为独立入口存在，用户需要深度对话时可从首页结果页再进入。
- 首页展示的“回答/概览”不做大模型生成，先用检索结果生成确定性摘要，避免新增幻觉风险。

## 建议步骤

1. 重写首页搜索组件的提交行为：从 `router.push('/chat?...')` 改为首页内调用文献检索接口。
2. 按截图结构做首页内研究工作台：左侧导航、中央问题与概览、右侧 References、底部搜索框。
3. 保留搜索模式和文献范围选项：快速、综述、深度；全部、中文、英文。
4. 为空结果、请求失败和加载状态补明确提示。
5. 运行前端构建验证，确认页面类型和打包通过。

## 不做内容

- 不删除 `pipeline` 页面。
- 不改聊天页 Agent 协议。
- 不新增依赖。
- 不改后端接口、数据库或文献源策略。

## 验收标准

- 在首页输入问题并提交后，页面不跳转到 `/chat`。
- 首页中部显示本次检索问题和基于结果的概览。
- 右侧显示 References 文献卡片。
- 未检索到结果时显示“暂无相关文献”，而不是空白。
- 前端构建通过。
