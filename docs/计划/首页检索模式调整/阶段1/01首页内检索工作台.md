# 阶段1 01：首页内检索工作台

## 目标

用最小改动把首页从“搜索后跳转聊天页”调整为“搜索后在首页展示结果”。这一阶段只验证交互方向，不做复杂 AI 总结和多页面联动。

## 修改范围

- `frontend/src/components/HomeSearchLanding.tsx`
- 必要时只补充使用已有类型和 API 封装，不新增依赖。

## 实施要点

1. 移除首页搜索提交中的 `/chat` 跳转。
2. 提交后调用 `searchLiterature`。
3. 增加状态：
   - 当前输入问题
   - 当前已提交问题
   - 检索模式
   - 文献范围
   - 文献结果
   - 加载状态
   - 错误信息
4. 参考截图组织布局：
   - 左侧：品牌、New Search、Home、登录状态。
   - 中央：用户问题气泡、检索路径、研究概览。
   - 右侧：References 列表。
   - 底部：固定或粘性的搜索输入框。
5. 中文文案使用 UTF-8 重写，避免现有乱码继续扩散。

## 验证方式

- 首页输入问题后 URL 不变为 `/chat`。
- 请求成功时可以看到文献结果。
- 请求失败时有错误提示。
- 空结果时展示“暂无相关文献”。
