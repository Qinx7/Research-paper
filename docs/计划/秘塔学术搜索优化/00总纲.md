# 秘塔式学术搜索与学术对话优化总纲

## 1. 需求理解

目标是参考秘塔 AI 搜索当前公开页面中体现出来的“学术搜索方式”和“搜索即研究”的交互思路，优化本项目现有的：

- 学术搜索功能；
- AI 学术对话 Agent；
- 搜索结果到回答结果之间的证据链路。

本次不是简单增加几个搜索源，而是要提升：

1. 搜索入口的表达能力；
2. 检索任务的组织方式；
3. 学术结果的筛选与排序；
4. 搜索结果进入 AI 回答时的证据利用质量。

---

## 2. 对秘塔当前公开能力的观察

基于当前公开页面可见信息，秘塔学术搜索至少体现出以下产品思路：

### 2.1 搜索范围明确分层

页面可见：

- 全网
- 文库
- 学术

说明它不是把“学术搜索”当作一个普通关键词搜索，而是把搜索域单独建模。

### 2.2 学术搜索支持进一步细分

页面可见：

- 所有文献
- 中文库
- 英文库

以及学术过滤项：

- PubMed
- 北大核心
- 中科院分区
- JCR
- SCIE
- EI

说明其学术搜索不仅有“搜到”，还强调：

- 数据源属性；
- 期刊/数据库质量；
- 学术评价体系；
- 中英文文献分流。

### 2.3 搜索模式不止一种

页面可见：

- 简洁深入
- 深度研究

以及文案：

- 多线迭代追搜直至逻辑闭环
- 先思考框架，再搜索整合资料
- 旧版，搜索信息，再扩展延伸

这说明它把搜索过程分成不同工作模式，而不是单轮关键词命中。

### 2.4 搜索与知识沉淀打通

页面可见：

- 上传文件
- 专题
- 知识岛

说明其学术搜索不仅面向公开网页，还在逐步形成：

- 用户上传资料作为补充语料；
- 项目/专题式知识组织；
- 搜索结果的长期复用。

### 2.5 搜索结果服务于回答，而不是止于列表

从整体文案和“深度研究”表达来看，搜索结果的目标不是单纯返回文献列表，而是：

- 聚合证据；
- 形成研究框架；
- 帮助回答复杂问题；
- 支持多轮追问。

---

## 3. 当前项目现状

现有项目已经具备以下基础：

### 3.1 学术搜索

- OpenAlex / Semantic Scholar / CNKI / 维普多源检索；
- 基本关键词提取；
- 去重与简单排序；
- 向量检索能力雏形；
- 文献结果可进入聊天上下文。

### 3.2 AI 学术对话

- 支持流式聊天；
- 可切换“学术检索”开关；
- 聊天时可注入搜索结果；
- 有向量库补充检索逻辑。

### 3.3 当前主要短板

与秘塔式能力相比，当前缺口主要在：

1. 检索意图建模太弱；
2. 没有显式“学术搜索模式”；
3. 缺少数据库/质量维度筛选；
4. 缺少中英文库分流；
5. 结果排序仍偏简单；
6. 聊天 Agent 对证据的利用方式偏粗；
7. 缺少“先分析问题，再组织检索”的规划步骤；
8. 缺少对项目私有资料/上传资料的整合检索闭环。

---

## 4. 建议的总体优化方向

### 4.1 把“学术搜索”从单次查询升级为“检索任务”

建议将学术搜索拆成明确阶段：

1. 问题理解；
2. 检索意图识别；
3. 检索策略生成；
4. 多源并行召回；
5. 结果去重与打分；
6. 证据摘要与回答。

### 4.2 增加搜索模式

建议首批至少提供 3 种模式：

- 快速检索：返回文献列表和简短摘要；
- 学术综述：聚合多篇文献，输出研究现状与趋势；
- 深度研究：先生成检索框架，再多轮扩搜，最后形成结构化回答。

### 4.3 增加学术过滤维度

建议引入可选过滤：

- 中文库 / 英文库；
- 数据源选择；
- 年份区间；
- 引用量阈值；
- 期刊/会议质量标签；
- 是否优先核心文献。

### 4.4 优化排序逻辑

当前排序建议从“简单去重后按引用量或已有顺序”升级为综合分：

- 主题相关性；
- 语义相似度；
- 引用影响力；
- 年份新近性；
- 来源质量；
- 是否适合作为当前问题证据。

### 4.5 优化聊天 Agent 的证据利用方式

当前聊天更像“把检索结果扔给模型”。建议升级为：

- 先把用户问题分类；
- 再决定是否检索、检索哪些源、检索多少轮；
- 把证据整理成结构化上下文；
- 回答时强制模型区分：
  - 直接证据；
  - 推断结论；
  - 未证实内容。

### 4.6 引入项目私有资料检索

后续可将：

- 用户上传文件；
- 项目成果；
- 开题设计；
- 历史论文草稿；

一起纳入“项目知识上下文”，形成类似秘塔“专题 / 上传文件”的项目内检索。

## 5. 推荐分阶段实施

### 阶段 1：学术搜索能力升级

目标：

- 让搜索本身更像“学术搜索”，而不是普通关键词召回。

建议内容：

- 搜索模式建模；
- 搜索过滤条件；
- 中英文库分流；
- 结果综合排序；
- 搜索结果结构扩充。

### 阶段 2：AI 学术对话 Agent 升级

目标：

- 让对话 Agent 具备“先想后搜、搜后总结、带证据回答”的能力。

建议内容：

- 查询意图分类；
- 检索规划；
- 多轮扩搜；
- 证据整理；
- 引用更严格。

### 阶段 3：项目知识闭环

目标：

- 把公开学术检索、项目私有资料和成果资料打通。

建议内容：

- 上传资料进入向量库；
- 搜索时同时检索公开文献与项目私有资料；
- 对话中区分“外部文献证据”和“项目内证据”。

---

## 6. 默认建议优先级

如果只做一轮高价值优化，建议优先顺序是：

1. 学术搜索模式 + 过滤条件；
2. 聊天 Agent 的检索规划与证据注入；
3. 结果排序升级；
4. 项目私有资料整合。

---

## 7. 需要确认的关键事项

1. 这轮优化你更想优先做：
   - 搜索页能力增强；
   - AI 学术对话增强；
   - 两者一起做。
2. 是否接受先做“公开文献搜索 + 对话增强”，把“项目私有资料整合”放到下一阶段。
3. 搜索模式首批是否按：
   - 快速检索；
   - 学术综述；
   - 深度研究
   这 3 种来设计。
