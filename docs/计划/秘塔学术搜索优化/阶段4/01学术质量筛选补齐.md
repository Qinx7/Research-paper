# 学术质量筛选补齐

## 1. 需求理解

当前系统已经支持：

- 搜索模式；
- 中英文库分流；
- 来源选择；
- 基础排序与部分质量标记。

但以下筛选项还没有形成真正可执行的能力：

- PubMed；
- IEEE；
- ACM；
- 北大核心；
- JCR；
- EI；
- 中科院分区；
- 开放获取。

本轮目标不是一次性接入所有权威目录，而是先把这些筛选从“界面占位”升级为“后端可执行、结果可解释”。

## 2. 关键假设

1. 本轮允许区分“真筛选”和“推断筛选”。
2. 前端可以明确标注“推断”，避免用户误解为官方认证结果。
3. 本轮不引入长期维护成本很高的完整目录库。

## 3. 实施边界

### 3.1 真筛选

- `PubMed`：新增真实文献源；
- `开放获取`：基于 OpenAlex 元数据做真实过滤；
- `年份 / 引用量 / 中英文库 / 来源选择`：统一纳入同一套筛选结构。

### 3.2 推断筛选

- `IEEE / ACM`：根据 venue、publisher、来源域名等做标签推断；
- `EI / JCR / 中科院分区 / 北大核心`：先基于来源与 venue 规则打标签，再允许按标签过滤。

说明：

- 当前版本接受“通过其他来源识别 IEEE / ACM 文献”的实现方式，不把 `IEEE Xplore / ACM DL` 官方 API 接入作为本轮前置条件。

### 3.3 本轮不做

- IEEE Xplore / ACM DL 官方 API 接入；
- JCR / 中科院 / 北大核心 / EI 权威目录的全量导入；
- 商业数据库授权和长期目录维护后台。

## 4. 建议实现

### 4.1 后端

新增统一质量筛选结构，例如：

- `source_indexes`
- `open_access_only`
- `min_citations`
- `year_from`
- `year_to`
- `quality_tags`

并在检索归一层补充统一字段：

- `is_open_access`
- `quality_flags`
- `quality_inference`

过滤顺序建议固定为：

1. 来源过滤；
2. 年份 / 引用量 / 开放获取；
3. 质量标签过滤；
4. 最终排序。

### 4.2 PubMed

新增 PubMed client，并统一映射为标准结果结构：

- title
- authors
- year
- venue
- abstract
- url
- source
- is_open_access

### 4.3 前端

只展示后端真实支持的过滤项，并显式区分：

- 真筛选；
- 推断筛选。

结果卡片中同步展示命中的质量标签，便于用户理解筛选依据。

## 5. 风险与控制

- 风险：用户把“推断标签”理解为“权威认证标签”。
  - 控制：UI 与返回字段统一显示“推断”。
- 风险：PubMed 接入后与现有源字段不一致。
  - 控制：统一在后端归一层做字段映射。
- 风险：规则标签误标率高。
  - 控制：第一版使用保守规则，不做激进猜测。

## 6. 验收标准

1. 前端筛选项能被后端真实消费。
2. `PubMed` 可作为真实来源参与检索与过滤。
3. `开放获取` 可基于真实元数据过滤。
4. `IEEE / ACM / EI / JCR / 中科院分区 / 北大核心` 至少形成可解释的标签筛选链路。
5. 搜索结果卡片能展示质量标签依据。
