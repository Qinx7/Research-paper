# 阶段 2：工具条与标尺感增强

## 目标

在阶段 1 的页面式编辑区基础上，增加轻量工具条和标尺感，让写作区更接近 Word/WPS 的工作氛围。

本阶段仍然不做假富文本功能，只展示真实可用操作和状态。

## 修改范围

- 修改：`frontend/src/app/writing/page.tsx`

## 具体改动

1. 调整顶部工具区。

   将当前顶部操作区整理为更像文档编辑器的工具栏：

   - 左侧显示草稿标题和当前章节；
   - 中间可显示保存状态；
   - 右侧保留真实操作：续写、复制；
   - 保存按钮仍可保留在底部，避免顶部按钮过多。

2. 增加水平标尺视觉。

   在纸张上方加入轻量标尺条：

   - 不具备拖动功能；
   - 用刻度和边距提示增强文档感；
   - 不影响编辑。

3. 证据状态摘要进入页面工具区。

   保留详细证据标签，同时在状态栏显示简要信息：

   - 有项目设计；
   - 有/缺成果材料；
   - 有/缺引用依据。

4. 增强底部状态栏。

   状态栏展示：

   - 字数；
   - 草稿进度；
   - 当前章节状态；
   - 保存状态；
   - 页面视图。

## 不做事项

- 不增加无效格式按钮；
- 不做真正标尺拖拽；
- 不做页面缩放；
- 不做字体选择；
- 不做富文本样式保存。

## 验证方式

1. 运行构建：

   ```powershell
   cd "D:\learning agent\frontend"
   npm.cmd run build
   ```

2. 人工检查：

   - 工具条看起来像文档编辑环境；
   - 标尺不遮挡内容；
   - 操作按钮都是真实可用能力；
   - 状态栏信息清晰；
   - 页面滚动正常。

## 完成标准

- 编辑区整体更接近 Word/WPS 的页面工作台；
- 没有新增假按钮；
- 构建通过。
