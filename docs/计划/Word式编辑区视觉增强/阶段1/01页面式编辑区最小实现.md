# 阶段 1：页面式编辑区最小实现

## 目标

把 `/writing` 中间编辑区从普通 `textarea` 视觉，改成类似 Word/WPS 的“灰色工作区 + 白色纸张”页面视图。

本阶段只做最小实现，重点验证视觉方向，不改变数据结构、不新增依赖。

## 修改范围

- 修改：`frontend/src/app/writing/page.tsx`

## 具体改动

1. 调整中间滚动区域背景。

   当前中间区域是直接 `px-12 py-10` 包裹文章内容。阶段 1 改成文档软件式工作台：

   - 外层背景使用浅灰或当前主题的低饱和背景；
   - 保持垂直滚动；
   - 内容居中。

2. 新增纸张容器。

   在 `article` 外或内部增加白色纸张视觉：

   - 最大宽度接近 A4 页面比例；
   - 最小高度模拟一页文档；
   - 使用边框和阴影；
   - 内边距模拟页边距。

3. 保留章节标题、证据标签和风险提示。

   这些信息仍放在纸张顶部，作为当前章节的文档头部信息。

4. 调整正文输入区。

   `textarea` 继续保留，但视觉上融入纸张：

   - 背景透明；
   - 无边框；
   - 字号接近论文正文；
   - 行距更接近文档编辑；
   - 最小高度随纸张增大。

5. 增加页面底部状态条。

   在中间编辑区底部或纸张下方显示：

   - 当前字数；
   - 当前章节状态；
   - 是否未保存；
   - 页面视图提示。

## 不做事项

- 不做富文本；
- 不做真实分页；
- 不做格式按钮；
- 不改右侧面板；
- 不改 API。

## 验证方式

1. 运行构建：

   ```powershell
   cd "D:\learning agent\frontend"
   npm.cmd run build
   ```

2. 人工检查：

   - `/writing` 中间区域呈现类似 Word/WPS 的页面视图；
   - 编辑、保存、续写、复制仍可用；
   - 切换章节后内容正常切换；
   - 第 4/5 章风险提示仍显示；
   - 右侧面板不被挤坏。

## 完成标准

- 页面视觉明显从“文本框”变成“文档页”；
- 所有原有写作操作保留；
- 构建通过。
