# 阶段 5：PC 键盘操作与编辑反馈

## 目标

项目主打 PC 用户，论文写作页需要补充桌面编辑器常见的键盘操作反馈。本阶段先做最小可用能力：`Ctrl+S` 保存当前章节。

## 修改范围

- 修改：`frontend/src/app/writing/page.tsx`

## 具体改动

1. 增加 `Ctrl+S` / `Cmd+S` 保存快捷键。

   - 在写作页监听键盘事件；
   - 阻止浏览器默认“保存网页”行为；
   - 当前有草稿且未处于保存中时，调用现有 `handleSave`；
   - 不新增保存接口，不改后端。

2. 增强底部状态反馈。

   - 在页面状态栏显示 `Ctrl+S 保存`；
   - 保留字数、进度、缩放、章节状态、证据状态。

## 不做事项

- 不做自动保存；
- 不做复杂快捷键系统；
- 不做快捷键自定义；
- 不做富文本快捷键；
- 不改后端。

## 验证方式

1. 运行构建：

   ```powershell
   cd "D:\learning agent\frontend"
   npm.cmd run build
   ```

2. 人工检查：

   - 在 `/writing` 编辑内容后按 `Ctrl+S` 能触发保存；
   - 浏览器不会弹出保存网页；
   - 没有草稿或保存中时不会重复提交；
   - 状态栏能看到 `Ctrl+S 保存` 提示。

## 完成标准

- PC 写作场景支持常用保存快捷键；
- 构建通过；
- 不新增依赖。
