# 阶段 4：页面缩放与阅读宽度控制

## 目标

增加 PC 文档编辑器常见的页面缩放和阅读宽度控制，让用户可以根据屏幕大小选择更舒适的写作视图。

本阶段仍然不做富文本，不改变草稿保存结构。

## 修改范围

- 修改：`frontend/src/app/writing/page.tsx`

## 具体改动

1. 增加页面缩放状态。

   - 使用前端本地状态保存当前缩放比例；
   - 默认 100%；
   - 提供 90%、100%、110% 三档；
   - 只影响页面视觉，不影响保存内容。

2. 增加阅读宽度控制。

   - 提供“标准 / 宽版”两档；
   - 标准接近 A4 文档页；
   - 宽版适合大屏显示更多内容。

3. 将控制入口放在顶部工具条。

   - 控制项必须真实生效；
   - 不加入富文本样式按钮；
   - 保持当前 `续写`、`复制` 操作。

4. 状态栏显示当前缩放。

   - 底部状态栏增加缩放比例；
   - 便于用户知道当前视图状态。

## 不做事项

- 不做浏览器级缩放；
- 不做真实分页；
- 不做字体选择；
- 不做富文本；
- 不改后端。

## 验证方式

1. 运行构建：

   ```powershell
   cd "D:\learning agent\frontend"
   npm.cmd run build
   ```

2. 人工检查：

   - 点击不同缩放比例，纸张视觉大小变化；
   - 切换标准/宽版，纸张宽度变化；
   - 输入、保存、续写、复制不受影响；
   - 状态栏显示当前缩放。

## 完成标准

- PC 用户可以调整文档视图舒适度；
- 无新增依赖；
- 构建通过。
