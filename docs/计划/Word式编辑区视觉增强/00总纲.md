# Word 式编辑区视觉增强总纲

## 1. 需求理解

当前论文写作页 `/writing` 的中间编辑区仍然是普通 `textarea`，虽然已经具备章节、证据、引用、交付面板，但写作体验更像“后台表单”，不像 Word 或 WPS 的论文编辑环境。

本方案采用“方案 1”：先做 Word/WPS 式页面视觉增强，不引入富文本编辑器，不改变后端保存结构，不新增依赖。目标是让用户进入写作页后看到类似文档软件的页面工作区：

- 灰色或浅色工作台背景；
- 中央 A4 纸张视觉；
- 纸张阴影、页边距、标题区；
- 顶部轻量工具条；
- 类似标尺和页面状态栏；
- 保留当前纯文本编辑、保存、AI 续写、复制、合规检查、导出流程。

## 2. 关键假设

- 本轮只做视觉与交互体验增强，不实现真正富文本。
- 当前 `textarea` 继续作为编辑主体，避免破坏保存和 AI 生成逻辑。
- 论文内容仍按章节保存为纯文本。
- 不新增依赖，不引入 TipTap、ProseMirror、Slate、Quill 等编辑器。
- 页面视觉以“论文写作工作台”为主，不做完整 WPS 功能复刻。

## 3. 不做事项

本方案明确不做：

- 不做真正分页排版；
- 不做富文本加粗、斜体、标题样式写入；
- 不做表格、图片、脚注、批注；
- 不改数据库；
- 不改后端导出逻辑；
- 不改 AI 生成接口；
- 不恢复之前已删除的无效格式按钮。

## 4. 推荐设计

### 中间区域

将当前中间编辑区改为“文档工作台”：

- 外层是可滚动的浅灰工作区；
- 中间是一张固定最大宽度的白色纸张；
- 纸张具有 A4 比例感、阴影、边框；
- 纸张内部保留章节标题、证据标签、风险提示；
- 文本编辑区使用更像论文正文的字体、字号、行距和页边距。

### 顶部工具条

顶部工具条不做假富文本，只放真实可用功能：

- 当前章节路径；
- 保存状态；
- 续写；
- 复制；
- 保存；
- 导出入口可保留在底部或右侧交付面板。

### 页面状态栏

编辑区底部增加轻量状态信息：

- 字数；
- 当前章节状态；
- 草稿进度；
- 页面视觉提示，如“页面视图”。

### 右侧面板

右侧 `建议 / 引用 / 交付` 保持不变，只需保证中间编辑区变成页面式后布局仍稳定。

## 5. 分阶段实施

### 阶段 1：页面式编辑区最小实现

目标：

- 将当前普通编辑区改成 Word/WPS 类似的页面工作区；
- 保留所有现有功能；
- 不新增工具按钮；
- 先保证视觉方向正确。

修改重点：

- `textarea` 外层增加纸张容器；
- 工作区背景改成文档软件常见的灰色画布；
- 纸张增加阴影、边框、内边距；
- 正文行距、字号、字体更接近论文写作；
- 底部状态栏显示字数和章节状态。

### 阶段 2：工具条与标尺感增强

目标：

- 在不做假功能的前提下增强文档软件感；
- 增加轻量标尺视觉；
- 工具区更像“可写作的文档编辑器”，不是普通网页表单。

修改重点：

- 顶部增加文档标题区和工具分组；
- 增加水平标尺视觉；
- 纸张顶部显示当前章节标题；
- 状态栏显示草稿进度、保存状态、证据状态摘要。

### 阶段 3：移动端与窄屏适配

目标：

- 保证小屏幕下写作页不横向挤爆；
- 右侧面板可保持现状或后续折叠；
- 中间纸张宽度自适应。

修改重点：

- 页面容器设置 `max-width` 与 `min-width` 边界；
- 纸张在小屏下降低内边距；
- 三栏布局如已有问题，单独再拆阶段处理。

## 6. 建议修改文件

- `frontend/src/app/writing/page.tsx`

暂不建议新增组件文件。当前写作页已经集中管理工作台布局，本轮视觉增强可以先在单文件内完成，后续若继续扩展富文本或复杂工具条，再拆组件。

## 7. 验收标准

- `/writing` 中间编辑区看起来像文档软件的页面视图，而不是普通文本框；
- 保存、续写、复制、证据标签、风险提示、右侧面板功能不回退；
- 不出现无效富文本按钮；
- 不新增依赖；
- `npm.cmd run build` 通过。
