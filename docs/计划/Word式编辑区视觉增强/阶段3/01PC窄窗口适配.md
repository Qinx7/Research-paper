# 阶段 3：PC 窄窗口适配

## 目标

本项目主打 PC 用户，因此本阶段不做移动端优先设计，而是保证常见 PC、笔记本和浏览器小窗口下 `/writing` 仍可用。

核心原则：

- 正常 PC 宽度尽量保留左侧章节栏、中间文档页、右侧证据面板；
- 只有窗口非常窄时，才隐藏次要侧栏；
- 不为了移动端牺牲 PC 工作台信息密度。

## 修改范围

- 修改：`frontend/src/app/writing/page.tsx`

## 具体改动

1. 三栏布局改为 PC 优先。

   - `xl` 及以上保留三栏；
   - `lg` 保留左侧章节栏和中间编辑区；
   - 更窄窗口只保留中间编辑区。

2. 纸张容器保留自适应。

   - 页面宽度继续使用 `max-width`；
   - 窄窗口下仅降低纸张内边距；
   - 不做移动端专用交互。

3. 顶部和底部工具区保留换行能力。

   - 小窗口下按钮不挤出屏幕；
   - 正常 PC 下保持工具栏信息密度。

## 不做事项

- 不做移动端抽屉；
- 不做移动端专用导航；
- 不新增依赖；
- 不改后端。

## 验证方式

1. 运行构建：

   ```powershell
   cd "D:\learning agent\frontend"
   npm.cmd run build
   ```

2. 人工检查：

   - PC 大屏保留三栏；
   - 常见笔记本宽度尽量保留右侧面板；
   - 小窗口下不横向挤爆；
   - 纸张编辑区仍可正常输入。

## 完成标准

- PC 主场景信息密度不被移动端适配牺牲；
- 窄窗口有基本兜底；
- 构建通过。
