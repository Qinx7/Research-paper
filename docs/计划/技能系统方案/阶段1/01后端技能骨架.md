# 阶段1：后端技能骨架最小实现

## 1. 目标

先在后端建立一个最小可用的 skill 系统骨架，不改太多现有业务逻辑。

## 2. 建议新增文件

```text
backend/app/skills/
├── models.py
├── registry.py
├── executor.py
├── __init__.py
```

## 3. 最小模型

建议 `SkillDefinition` 包含：

- `id`
- `name`
- `description`
- `domain`
- `input_schema`
- `output_schema`
- `handler`
- `guards`
- `enabled`

## 4. 最小执行流程

1. 前端或 API 指定 `skill_id`
2. Registry 找到技能定义
3. Executor 校验输入
4. 调用 handler
5. 执行 guards
6. 返回标准输出

## 5. 不做事项

- 不做数据库注册
- 不做用户自定义技能
- 不做前端技能管理页
