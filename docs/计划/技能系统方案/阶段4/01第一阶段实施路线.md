# 阶段4：第一阶段实施路线

## 1. 推荐落地顺序

### 第一步

新增 skill 骨架：

- `SkillDefinition`
- `SkillRegistry`
- `SkillExecutor`

### 第二步

把写作页 AI 续写动作内部接到：

- `paper.chapter_draft`
- `paper.chapter_grounding`

### 第三步

把答辩 PPT 生成动作内部接到：

- `ppt.defense_outline`
- `ppt.defense_render`

### 第四步

为现有 workflow 增加 skill metadata 记录：

- `skill_id`
- `skill_version`
- `guard_result`

## 2. 建议验证方式

1. 写作页 AI 续写仍然可用
2. grounding guard 仍然生效
3. 答辩 PPT 生成仍然可用
4. workflow 记录中能看到对应 skill_id

## 3. 风险点

### 3.1 过度抽象

如果一开始就试图把所有 Agent 都 skill 化，容易拖慢进度。

### 3.2 Guard 与 Skill 分层混乱

Guard 必须是 Skill 的附属约束，不应继续散落到多个 API 里重复写。

### 3.3 前端过早暴露技能选择

第一阶段不建议让用户直接看到 skill 层。
