## 3. 数据库迁移检查

如果阶段 2 增加字段，需要执行：

```powershell
cd backend
alembic upgrade head
alembic current
```

验收点：

1. `agent_workflow_runs` 新字段存在。
2. `agent_workflow_steps` 新字段存在。
3. 旧记录仍可查询。
4. 新记录能写入 `result_ref` 和 diagnostics。

## 4. 前端构建检查

```powershell
cd frontend
npm.cmd run build
```

验收点：

1. 不新增 `/workflows` 用户页面入口。
2. 首页历史检索恢复不重新生成。
3. 研究页打开已有结果不重新生成。
4. 写作页打开已有章节不重新生成。
5. PPT / Deck 入口仍可正常使用。

## 5. 人工验收路径

### 5.1 首页检索

1. 在首页发起一次检索。
2. 确认结果、总结、来源诊断正常显示。
3. 点击左侧历史记录。
4. 确认直接恢复结果，不重新检索。

### 5.2 研究页

1. 从已有检索进入研究页。
2. 生成研究方向。
3. 重新打开同一记录。
4. 确认不会重复生成方向。

### 5.3 写作页

1. 打开已有项目草稿。
2. 生成或查看章节。
3. 执行整篇审查。
4. 确认引用依据和内部资料依据仍可追踪。

### 5.4 PPT / Deck

1. 从项目设计或论文草稿生成 HTML Deck。
2. 预览和下载链接可用。
3. workflow 记录中能看到 artifact 引用。

## 6. 不通过条件

出现以下情况视为本阶段不通过：

1. 点击历史记录仍触发重新生成。
2. workflow 记录写入失败导致主业务失败。
3. 普通用户前端重新出现执行记录入口。
4. skill route 缺失时返回空结果而不是明确错误。
5. 新增字段没有 Alembic 迁移。
