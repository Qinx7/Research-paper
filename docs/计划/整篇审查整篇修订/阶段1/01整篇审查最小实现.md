# 阶段1 01 整篇审查最小实现

## 1. 目标

先补一个最小可用的整篇审查能力，不做复杂语义重写，只先把“整篇有哪些明显问题”稳定输出出来。

## 2. 后端建议

新增：

- `paper.full_review_pass`

输入：

- `draft_title`
- `full_text`
- `chapter_summaries`
- `citations`
- `evidence_context`

输出：

- `passed`
- `summary`
- `issues`
- `focus_areas`
- `chapter_flags`

## 3. 首批审查规则

建议先覆盖这些：

1. 整篇篇幅不均衡
2. 相邻章节衔接弱
3. 摘要/结论与正文不呼应
4. 第一章文献支撑不足
5. 第五章实验结构不完整
6. 重复表述过多
7. 术语前后不一致

## 4. 接口

- `POST /api/drafts/{draft_id}/review-full`
