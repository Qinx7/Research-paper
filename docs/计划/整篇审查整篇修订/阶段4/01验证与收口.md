# 阶段4 01 验证与收口

## 1. 后端验证

至少验证：

1. `paper.full_review_pass` 注册成功
2. `paper.full_revision_apply` 注册成功
3. `review-full` 接口返回结构稳定
4. `revise-full` 接口返回结构稳定
5. LLM 修订优先、规则型 fallback 生效

## 2. 前端验证

至少验证：

1. `/writing` 构建通过
2. 整篇审查按钮可点击
3. 整篇修订按钮可点击
4. 审查结果面板展示正常
5. 修订结果面板展示正常
6. 修订状态与前后对照展示正常

## 3. 端到端验收

从用户视角应能完成：

1. 进入 `/writing`
2. 选择草稿
3. 生成完整初稿
4. 查看整篇连续正文
5. 发起整篇审查
6. 查看整篇问题
7. 发起整篇修订
8. 查看整篇修订结果
9. 保存整篇

## 4. 完成标准

只有在以下条件同时满足时，才算真正解决前面的问题：

1. `/writing` 已经是唯一主写作页
2. 用户可以在同页进行整篇生成与整篇修改
3. 不再必须逐章操作才能形成整篇初稿
4. 整篇审查和整篇修订都可在当前页完成
