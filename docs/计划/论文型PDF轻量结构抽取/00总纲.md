# 论文型 PDF 轻量结构抽取总纲

## 1. 目标

在现有全文解析栈基础上，为 `scholarly_pdf` 增加一版**轻量规则抽取**能力，优先补 `title / abstract / references` 三类结构化信息，不引入 `GROBID` 外部服务。

## 2. 现状

当前已经具备：

1. PDF 多策略解析链：`pypdf -> pdfplumber -> ocr`
2. `document_kind` 区分：
   - `scholarly_pdf`
   - `general_pdf`
3. `structured_fields` 预留：
   - `title`
   - `abstract`
   - `sections`
   - `references`

但目前这些字段还只是“预留位”，没有真正抽出内容。

## 3. 本轮范围

本轮做：

1. 对 `scholarly_pdf` 尝试规则抽取标题
2. 对 `scholarly_pdf` 尝试规则抽取摘要
3. 对 `scholarly_pdf` 尝试定位参考文献区
4. 把抽取结果写入 `meta`

本轮不做：

1. 真实 `GROBID` 接入
2. 章节层级精确解析
3. 参考文献逐条规范化
4. 页码级定位

## 4. 推荐策略

### 4.1 标题抽取

优先规则：

1. 取开头若干行中最长、最像标题的一行
2. 跳过明显的 `Abstract / Keywords / DOI / 作者信息`
3. 优先选英文论文风格中首段大标题

### 4.2 摘要抽取

优先规则：

1. 定位 `Abstract` 或 `摘要`
2. 抽取到 `Keywords / 关键词 / Introduction / 1.` 之前
3. 长度限制在合理范围内

### 4.3 参考文献区定位

优先规则：

1. 定位 `References / Reference / 参考文献`
2. 提取该标记后正文剩余文本
3. 暂只保存整段，不拆成条目

## 5. 数据结构建议

在 `ParsedDocument.meta` 中补充：

1. `structured_content`
   - `title`
   - `abstract`
   - `references_text`
2. `structured_confidence`
   - `title`
   - `abstract`
   - `references`

## 6. 风险

1. 规则抽取对不同版式鲁棒性有限
2. OCR 结果较乱时，摘要和参考文献定位可能误判
3. 不能把这版规则抽取包装成正式学术结构解析

## 7. 成功标准

1. 至少能对一部分论文型 PDF 抽出标题
2. 有 `Abstract / 摘要` 的 PDF 能抽出摘要
3. 有 `References / 参考文献` 的 PDF 能定位末尾参考文献区
4. 后端测试通过
