# 文献知识图谱方案总纲

## 1. 需求理解

当前系统已通过文献检索收集了大量 Paper 数据（标题、作者、年份、期刊、引用次数、摘要），但**只以列表形式展示**。研究生在撰写文献综述时需要：

- 快速理解文献之间的关系和脉络
- 识别核心论文、关键作者和研究热点
- 观察研究主题随时间的演进趋势
- 发现研究空白和潜在突破点

本次目标：基于**已有 Paper 数据**（无需新增外部依赖），构建交互式文献知识图谱可视化。

## 2. 关键设计决策

### 决策 1：不依赖外部引用数据，从现有字段派生关系

Paper 表没有 citation 关系字段，获取引用网络需要调用 OpenAlex API。**本次不引入外部依赖**，从已有字段派生图关系：

| 关系类型 | 数据来源 | 说明 |
|----------|----------|------|
| **关键词共现** | LLM 从 title + abstract 提取关键词 | 两篇论文共享关键词 → 产生边 |
| **作者合作** | authors 字段解析 | 同一作者出现在多篇论文 → 产生边 |
| **期刊聚类** | venue 字段 | 同一期刊/会议的论文聚为一组 |
| **时间线** | year 字段 | 按年份组织论文，展示研究演进 |

### 决策 2：ECharts 做可视化，零新依赖

ECharts 已在前端项目？实际上需要安装 `echarts` 和 `echarts-for-react`。这是本次唯一的新依赖，但 ECharts 是国内项目标配。

### 决策 3：4 种视图，标签页切换

| 视图 | 图表类型 | 说明 |
|------|----------|------|
| **关系网络** | Force-directed graph | 论文节点 + 关键词/作者边，可拖拽探索 |
| **时间演进** | Timeline scatter | X 轴年份，Y 轴引用数，气泡大小=影响力 |
| **主题聚类** | Treemap / 旭日图 | 按关键词聚类的层级结构 |
| **引用排行** | Bar chart | Top N 高引论文 + 期刊分布 |

### 决策 4：LLM 提取关键词 → 持久化到 Paper 表

关键词提取通过 LLM 批量处理（一次请求处理所有论文标题+摘要）。提取结果保存到 `Paper.keywords` JSONB 字段，**不会丢失**。

**持久化策略**：
- `Paper` 模型新增 `keywords` JSONB 字段（`nullable=True`）
- 首次构建图谱时，检查每篇论文的 `keywords` 是否为空 → 空则调 LLM 提取 → 写入 DB
- 后续请求直接读取 `keywords`，不再重复调用 LLM
- 用户可手动编辑某篇论文的关键词（后续功能预留）

### 决策 5：ECharts 做可视化

使用 `echarts` + `echarts-for-react`。国内项目标配，中文友好，图表类型丰富。这是本次唯一的新依赖。

## 3. 实施步骤

### 阶段 1：后端图谱数据 API

**步骤 1.1：关键词提取 Agent 方法**
- 在现有 `literature_review_agent.py` 或新建独立方法
- 批量输入论文 title + abstract，LLM 提取关键词列表
- 返回 `{paper_id: [keyword1, keyword2, ...]}`

**步骤 1.2：图谱数据构建服务**
- 新建 `backend/app/services/knowledge_graph_service.py`
- 函数：`build_graph_data(papers, keywords_map) -> GraphData`
- 计算：节点、边、聚类、时间序列

**步骤 1.3：API 端点**
- 修改 `backend/app/api/literature.py`
- `GET /api/literature/{project_id}/knowledge-graph?view=network|timeline|clusters|impact`

### 阶段 2：前端可视化

**步骤 2.1：安装 ECharts**
- `npm install echarts echarts-for-react`

**步骤 2.2：KnowledgeGraph 组件**
- 新建 `frontend/src/components/KnowledgeGraph.tsx`
- 4 标签页视图切换
- 节点 hover 显示论文详情 tooltip
- 节点点击可打开论文链接

**步骤 2.3：集成到项目页**
- 修改 `frontend/src/app/projects/[id]/page.tsx`
- 新增"知识图谱"标签页或卡片
