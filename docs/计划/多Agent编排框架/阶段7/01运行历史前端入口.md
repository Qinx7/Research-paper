# 阶段 7：运行历史前端入口

## 1. 目标

为已经接入轻量多 Agent 编排框架的 workflow 提供前端可视化入口，让用户能查看每次运行的状态、输入摘要、输出摘要、错误信息和节点执行明细。

本步骤只做只读展示，不新增 workflow 调度能力，不修改首页文献检索主交互。

## 2. 已实现范围

- 新增 `/workflows` 页面。
- 新增前端类型：
  - `AgentWorkflowRun`
  - `AgentWorkflowStep`
  - `AgentWorkflowRunDetail`
- 新增前端 API 封装：
  - `listAgentWorkflowRuns`
  - `getAgentWorkflowRun`
- 页面默认加载当前用户最近 30 条 workflow run。
- 支持点击单条 run 查看 step 时间线。
- 展示 run 级输入摘要、输出摘要、错误信息、状态和关联项目 ID。
- 展示 step 级输入摘要、输出摘要、错误信息和耗时。
- 首页“其他入口”菜单增加“执行记录”入口。
- 项目详情页 header 增加“执行记录”入口。

## 3. 当前可查看的 workflow 类型

- 首页文献检索：`home_literature_search`
- 论文章节生成：`paper_chapter_generation`
- 开题报告生成：`proposal_generation`
- 研究方向生成：`research_direction_generation`
- 项目设计生成：`project_design_generation`

## 4. 验证方式

- `frontend/npm.cmd run build`

## 5. 非本步骤范围

- 不新增 workflow 重跑、取消、恢复、删除功能。
- 不新增按项目筛选、按类型筛选、分页加载。
- 不修改后端记录结构。
- 不接入 PPT / 答辩 PPT workflow。

## 6. 下一步建议

继续扩大编排覆盖面时，优先接入 PPT / 答辩 PPT 生成任务；继续增强可观测性时，优先补筛选、分页和失败重试入口。
