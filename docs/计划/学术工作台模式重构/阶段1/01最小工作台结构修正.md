# 阶段 1：最小工作台结构修正

## 目标

快速把产品从“模块概览页”纠正为“固定侧边栏 + 工作区”的模式，优先解决用户指出的最大偏差：论文写作必须是编辑工作区，文献搜索只做文献检索。

## 修改范围

- `frontend/src/app/page.tsx`
- `frontend/src/app/chat/page.tsx`
- `frontend/src/app/writing/page.tsx`
- `frontend/src/components/chat/ChatSidebar.tsx`
- 必要时新增一个可复用的工作台侧边栏组件，但第一阶段优先复用现有 `ChatSidebar` 思路，避免大重构。

## 文献搜索最小改动

- 首页默认进入文献搜索体验。
- 保留当前学术检索输入、模式、范围、项目上下文等真实能力。
- 页面文案收束为“文献搜索”，避免把它描述成完整科研流程。
- 检索结果继续以证据卡片展示，后续阶段再精修卡片布局。

## 论文写作最小改动

- 撤掉当前 `/writing` 的“论文写作工作台概览 + 项目卡片”主形态。
- 首屏改成截图 3 的编辑器结构：
  - 左侧：项目信息、完成进度、章节结构。
  - 中间：当前章节标题、正文编辑/预览区域。
  - 右侧：AI 写作建议、引用、合规检查入口。
- 真实功能仍由 `PaperWorkflow` 提供，不在第一阶段重写生成逻辑。
- 如果没有项目或草稿，才显示空状态和进入完整流程按钮。

## 研究方向最小改动

- 保留当前 `/research` 的真实数据读取。
- 视觉上保持截图 2 的“左侧方向列表 + 右侧仪表盘”方向。
- 本阶段不深改趋势图和作者/会议等模拟面板，留到阶段 3。

## 验证方式

- 运行 `npm.cmd run build`。
- 手动打开 `/chat`，确认页面只围绕文献检索。
- 手动打开 `/writing`，确认进入的是论文编辑工作区，而不是概览卡片页。
- 手动打开 `/research`，确认路由仍正常。

## 风险

- `PaperWorkflow` 当前文件较大，直接深改容易引入生成、保存、合规检查回归。
- 因此第一阶段应只调整外层布局和默认展示状态，真实业务逻辑尽量保持原样。
