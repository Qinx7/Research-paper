# 学术工作台模式重构总纲

## 需求理解

本轮要纠正当前页面的信息架构与交互模式，使系统更接近参考截图中的 Scholar 工作台，而不是继续使用之前的“首页/模块概览卡片”式设计。

目标结构为三个固定模块：

- 文献搜索：只承担文献检索任务，形态接近 pasa-agent.ai、OpenAlex、Consensus 一类检索产品，并参考截图 1 的“对话式检索 + 文献结果卡片”。
- 研究方向：承担方向分析、趋势分析、热点和空白展示，形态参考截图 2 的“左侧方向列表 + 右侧分析仪表盘”。
- 论文写作：必须是可编辑的论文工作区，形态参考截图 3 的“左侧章节结构 + 中间编辑器 + 右侧 AI 建议/引用面板”。

## 当前偏差

- 论文写作页刚改成了“项目概览 + 进入工作流”的结构，但用户实际要的是直接进入编辑工作区。
- 文献搜索不应承担过多“研究流程入口”或“论文分析流程入口”的展示，只需要围绕文献检索和文献结果展开。
- 研究方向和论文写作应像同一套工作台中的不同模式，而不是各自独立的大型落地页。

## 关键假设

- 继续保留现有后端能力，不新增数据库结构，不新增依赖。
- 第一阶段优先改前端信息架构和页面布局，尽量复用当前聊天、研究方向、PaperWorkflow 的真实功能。
- 文献搜索仍可复用当前聊天 SSE 和学术检索接口，但 UI 需要弱化“聊天机器人”，强化“检索结果列表”。
- 论文写作需要尽量复用现有 `PaperWorkflow` 里的草稿、章节生成、合规检查和答辩 PPT 能力，但外观应改成编辑器工作区，而不是项目选择概览。

## 总体方案

采用统一 Scholar 工作台框架：

- 左侧固定深绿色导航栏，包含品牌、三个功能模块、近期会话、用户入口。
- 每个模块右侧使用对应工作区，而不是跳到风格完全不同的页面。
- 色彩沿用截图中的深绿、米白、纸张色、暗红强调色，不再做过度装饰。
- 首页默认进入文献搜索，不再展示单独营销式首页。

## 分阶段实施

- 阶段 1：最小工作台结构修正。统一侧边栏，首页进入文献搜索，论文写作改为编辑器工作区外观。
- 阶段 2：文献搜索结果体验修正。让检索页更像截图 1，只展示检索对话、状态、文献卡片、筛选和精准模式。
- 阶段 3：研究方向页面对齐截图 2。保留真实数据读取，把方向列表、统计卡、趋势图、热点/空白区做成工作台面板。
- 阶段 4：论文写作深度对齐截图 3。把 `PaperWorkflow` 的阶段 2 草稿能力前置为默认编辑体验，并将成果、合规、答辩能力作为侧边/顶部工具。

## 验收标准

- `/` 或 `/chat` 看起来是文献检索产品，不是项目流程首页。
- `/writing` 首屏就是论文编辑器工作区：左侧目录，中间正文编辑区，右侧建议/引用区。
- `/research` 保持方向分析仪表盘，不再像流程卡片。
- 三个模块共用一致的左侧导航和视觉语言。
- `npm.cmd run build` 通过。

## 需要确认

当前建议第一阶段先做最小修正：

- 首页直接重定向或渲染文献搜索工作台。
- 论文写作页撤掉刚才的项目概览样式，改成截图 3 的编辑工作区模式。
- 研究方向页暂只微调，不大重写。

确认后再开始代码修改。
