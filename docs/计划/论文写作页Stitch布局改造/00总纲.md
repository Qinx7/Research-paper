# 论文写作页 Stitch 布局改造总纲

## 需求理解

参考 Stitch 项目 `Literature Review Workspace` 中的 `Literature Review Editor` 页面，将当前论文写作页改造成更像专业文档编辑器的工作台。

Stitch 页面核心结构是：

- 顶部固定导航栏：品牌、页面入口、保存、导出、用户入口。
- 中间文档画布：白色纸张、浮动格式工具栏、标题输入、正文编辑区。
- 右侧 Sources 栏：文献来源搜索、来源卡片、添加来源入口。

当前项目的论文写作页核心能力是：

- 项目与草稿选择。
- 草稿章节目录与整篇完成度。
- 整篇生成、写作计划、大纲生成、摘要生成。
- 当前章节编辑、审查、修订。
- 整篇审查、整篇修订。
- 项目资料搜索、插入当前章节、跳转项目定位。
- 合规检查、问题确认、导出 DOCX/PDF。

本次目标是用 Stitch 的布局组织方式承载这些既有功能，不扩展新后端能力。

## 关键假设

- 本轮只改 `frontend/src/app/writing/page.tsx` 的页面结构与样式。
- 不改接口、不改数据库、不新增依赖。
- 保留当前所有按钮功能和状态逻辑，只调整展示位置。
- 页面主体验采用蓝白浅灰风格，与当前首页保持一致，不完全照搬 Stitch 的英文品牌与色值。
- 当前乱码文本如果已存在于文件中，本轮不做大范围文案修复，只在触及区域尽量使用正常中文。

## 改造方向

### 页面结构

采用三段式编辑器：

- 顶部栏：左侧返回/品牌与项目草稿信息，中间轻量入口，右侧保存、导出、用户状态。
- 中央画布：顶部浮动工具条，中间 Word 式论文纸张，章节按顺序在同一页连续编辑。
- 右侧 Sources/Assistant 栏：项目资料搜索、引用/证据来源、章节动作、审查修订状态、合规问题。

### 功能映射

- Stitch `Save` 对应当前 `handleSaveAll`。
- Stitch `Export` 对应当前 `handleDownload` 和格式选择。
- Stitch 浮动工具条对应当前章节操作快捷入口：生成计划、生成大纲、生成完整初稿、摘要、审查、修订、合规检查。
- Stitch `Sources` 对应当前项目资料搜索结果、当前章节引用、审查/合规证据提示。
- Stitch `Add Source` 对应当前“搜索项目资料 / 插入当前章节”。

## 成功标准

- 页面视觉上明显接近 Stitch：顶部栏、居中纸张、浮动工具条、右侧 Sources 栏。
- 当前所有已暴露功能仍能点击，原处理函数不丢失。
- 中间写作区保持同一页连续章节编辑，不回到多个模块分割。
- 页面整体固定在视窗内，内部区域滚动，避免整页乱滚。
- `npm.cmd run build` 通过。

## 不纳入本轮

- 不新增富文本编辑器依赖。
- 不实现真实加粗、斜体、引用插入富文本命令。
- 不调整后端生成逻辑。
- 不重构 `PaperWorkflow.tsx`，本轮优先改 `/writing` 独立页面。
