# 阶段1 01 编辑器骨架最小实现

## 目标

先用最小改动把 `/writing` 页面改成 Stitch 风格的编辑器骨架，验证整体方向。

## 范围

- 修改 `frontend/src/app/writing/page.tsx` 的 JSX 结构和 Tailwind class。
- 保留当前 state、effect、handler 与 API 调用。
- 将当前三栏布局调整为：
  - 顶部固定栏。
  - 左侧窄项目/章节导航栏。
  - 中间文档纸张编辑区。
  - 右侧 Sources/Assistant 栏。

## 具体步骤

1. 顶部栏替换当前页面头部。
2. 左侧栏保留项目选择、草稿选择、章节目录、进度。
3. 中央区域使用白色纸张容器承载连续章节 textarea。
4. 工具条放在纸张上方，承载生成、审查、保存等按钮。
5. 右侧栏先迁移“当前章节动作”和“项目资料搜索”。

## 验证

- `/writing` 页面能构建通过。
- 项目选择、草稿选择、章节定位仍可用。
- 中间章节内容仍可编辑。
- 保存整篇按钮仍调用原 `handleSaveAll`。

## 风险

- 当前文件较大，直接重排 JSX 容易误删功能按钮。
- 需要逐段迁移，避免一次性重写。
