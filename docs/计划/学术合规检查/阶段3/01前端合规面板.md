# 阶段 3 步骤 01：前端合规展示 + 用户确认

## 目标

在 PaperWorkflow 章节编辑器中展示合规检查结果，支持用户逐条确认/忽略 issues。

## 1. API 函数（修改 `frontend/src/lib/api.ts`）

```typescript
// 运行合规检查
export async function checkCompliance(draftId: string, enableAi = false) {
  const res = await fetch(`${API}/drafts/${draftId}/check-compliance?enable_ai=${enableAi}`, {
    method: "POST",
  });
  return res.json();
}

// 用户确认 issue
export async function confirmComplianceIssue(
  draftId: string,
  chapterKey: string,
  issueIndex: number,
  action: "accept" | "ignore" | "fixed"
) {
  const res = await fetch(`${API}/drafts/${draftId}/confirm-compliance`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ chapter_key: chapterKey, issue_index: issueIndex, action }),
  });
  return res.json();
}

// 获取合规状态
export async function getComplianceStatus(draftId: string) {
  const res = await fetch(`${API}/drafts/${draftId}/compliance-status`);
  return res.json();
}
```

### 状态徽章规则
- 🟢 绿色：0 error，0 warning
- 🟡 黄色：0 error，有 warning
- 🔴 红色：有 error
- ⚪ 灰色：未检查

### 用户确认流程
1. 章节生成 → 自动规则检查 → 徽章变红/黄
2. 用户点击 issue → 展开详情
3. 用户选择：
   - **确认通过** → issue 标记为 accepted，不计入扣分
   - **标记忽略** → issue 标记为 ignored，仍计入统计
   - **去修改** → 跳转到编辑模式对应位置
4. 可选：点击"深度检查"按钮 → 调用 AI → 新增 info 级别 issues

## 3. 验证

1. 无成果生成章节 → 编辑器中合规面板显示红色徽章
2. 展开 issues → 看到数据真实性 error + 章节标记 warning
3. 点击"确认通过" → issue 状态更新，徽章颜色可能变化
4. 点击"深度检查" → 显示 loading → 收到 AI issues
