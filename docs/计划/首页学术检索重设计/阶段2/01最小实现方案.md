# 阶段2 01 最小实现方案

## 目标

在不改后端的前提下，把首页检索体验改成“证据罗盘”样式，并让现有检索参数更早、更清楚地暴露给用户。

## 建议改动文件

- `frontend/src/components/PipelineWizard.tsx`
- 如需少量全局动效或纹理，修改 `frontend/src/app/globals.css`

## 组件结构建议

在 `PipelineWizard.tsx` 内部新增少量本地展示组件，避免拆太多文件：

- `SearchModeCard`：展示快速检索、学术综述、深度研究。
- `ScopePill`：展示所有文献、中文库、英文库。
- `EvidenceRoute`：展示问题理解、关键词抽取、跨库检索、证据评分、研究方向。
- `SearchStrategyPreview`：展示分析出的中英文关键词、方法、应用场景。
- `SourceSummaryBar`：在第 2 步结果区展示 OpenAlex、Semantic Scholar、CNKI、维普来源数量。

## 交互规则

### 首屏输入

- 用户输入自然语言研究问题。
- 可在输入前选择检索模式和文献范围。
- 高影响优先、最低引用量保留为简洁策略控件。
- 按钮文案从“开始分析”改为“启动证据检索”。

### 第 1 步完成后

- 继续展示分析结果，但从“分析结果”改成“检索策略草案”。
- 关键词用更像检索 token 的样式展示。
- 初步建议作为“Agent 计划”展示。

### 第 2 步检索

- 顶部展示当前检索策略。
- 检索结果先显示来源分布和总数。
- 文献卡片保留原 `PaperCard`，不扩大重构。

## 视觉方向

- 延续深绿、暖纸、酒红强调色。
- 首页主体使用“研究档案桌面”氛围：纸质背景、仪表盘卡片、细线分区。
- 不使用紫色渐变、不使用泛 AI 风卡片。
- 交互要显得像“专业研究工具”，不是营销落地页。

## 不做内容

- 不新增后端检索源。
- 不新增真正 OpenAlex 自动补全。
- 不新增实时流式 Agent 计划。
- 不改聊天页。
- 不改论文阶段。

## 测试计划

1. 运行 `npm.cmd run build`。
2. 手动打开首页，输入超过 10 个字的研究需求。
3. 选择三种检索模式之一，确认状态写入 `state.literatureSearchMode`。
4. 选择文献范围，确认状态写入 `state.literatureLibraryScope`。
5. 点击启动后进入第 2 步，确认后续检索按钮仍可触发 `handleStage2`。
