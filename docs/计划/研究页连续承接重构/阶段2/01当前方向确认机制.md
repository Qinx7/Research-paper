# 阶段2 01 当前方向确认机制

## 目标

建立“预览方向”和“当前方向”的明确区分，避免左侧点击即修改当前方案。

## 背景

用户已明确要求：

1. 左侧点击方向时先预览
2. 不要自动切换当前方向
3. 通过右侧按钮显式确认“设为当前研究方向”

## 状态设计

建议至少区分：

1. `previewDirectionId`
   当前右侧正在查看的方向
2. `currentDirectionId`
   当前项目被确认承接的方向

## 交互规则

1. 左侧点击某方向：
   只更新 `previewDirectionId`
2. 若预览方向不是当前方向：
   右侧显示“设为当前研究方向”按钮
3. 用户点击确认后：
   更新当前方向状态
4. 项目设计与开题材料区默认以当前方向为正式承接对象

## UI 提示建议

1. 当前方向卡片显示“当前方案”标识
2. 预览中但未确认的方向显示“预览中”或仅高亮
3. 右侧标题区显示当前是否已确认

## 最小实现要求

1. 不引入复杂全局状态
2. 先在研究页内部完成区分
3. 若已有后端保存入口可复用，则优先复用
4. 若当前项目已有 `direction_id` 承接逻辑，则尽量沿用

## 风险

1. 若当前方向与已有项目设计的绑定关系处理不清，可能导致展示错位
2. 若只是前端本地状态，不落项目承接，会导致刷新后丢失

## 验证点

1. 左侧切换方向时不会直接改变当前方向
2. 用户能明确知道哪个方向是当前方案
3. 当前方向确认后，后续承接区状态正确
