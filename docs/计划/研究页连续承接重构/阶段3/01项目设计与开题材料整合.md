# 阶段3 01 项目设计与开题材料整合

## 目标

把“项目设计”和“开题材料”从零散动作按钮，收敛成连续承接的内容区。

## 项目设计区

应体现：

1. 当前方向是否已有项目设计
2. 若未生成：
   显示“生成项目设计”主按钮
3. 若已生成：
   展示设计主题、摘要、最近生成状态
4. 提供“重新生成设计”与后续承接动作

## 开题材料区

统一归并：

1. 开题报告
2. 开题 PPT

每类材料至少区分：

1. 未生成
2. 生成中
3. 已生成可下载

## 交互规则

1. 开题材料区在项目设计之后展示
2. 若当前方向还没有项目设计，应弱化开题材料动作
3. 若已有项目设计，允许直接生成开题报告和开题 PPT
4. 下载入口统一放在当前区块内部

## 视觉要求

1. 像“研究方案承接结果”，不是工具栏按钮区
2. 使用状态卡片或分区说明替代单纯按钮堆叠
3. 每个区块都要让用户知道“当前是什么状态”

## 验证点

1. 项目设计区和开题材料区顺序清晰
2. 用户不看说明也能理解下一步可以做什么
3. 已生成的结果可直接查看或下载
