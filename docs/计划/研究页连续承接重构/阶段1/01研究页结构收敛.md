# 阶段1 01 研究页结构收敛

## 目标

把当前研究页右侧主体从“分析信息集合”重构成更完整的“研究方案详情”。

## 调整范围

涉及页面：

1. `frontend/src/app/research/page.tsx`

主要调整：

1. 保留左侧方向列表
2. 右侧顶部保留标题、研究领域标签、少量关键状态
3. 研究背景、研究问题、方法、预期成果重组为详情内容主轴
4. 评分与研究空白保留，但从“图表中心”降级为“辅助信息区”
5. 趋势图弱化，必要时改为更轻量的信息摘要

## 结构建议

右侧区块顺序建议：

1. 方向标题与当前状态
2. 研究背景摘要
3. 核心问题 / 方法 / 预期成果
4. 关键评分与研究空白
5. 项目设计承接区
6. 开题材料区

## 最小实现要求

1. 不新增页面
2. 不改路由
3. 不新增后端接口
4. 先通过重排和收敛现有区块完成结构转变

## 验证点

1. 用户第一眼看到的是研究方案主体，不是图表
2. 图表信息不再压过承接动作区
3. 页面仍可加载并切换不同方向
