# Zotero 同步 — 总纲

## 目标

实现本系统与 Zotero 文献库的双向同步。MVP 阶段先完成 **Zotero → 系统** 单向导入。

## 核心场景

1. 用户在 Zotero 中管理了大量文献 → 想导入到本系统用于开题/论文写作
2. 选择要同步的 Zotero 集合 → 一键导入 → 文献进入项目文献库
3. 后续可增量同步（Zotero 中新增的文献自动拉取）

## 架构概览

```
Zotero Web API (api.zotero.org)
        ↑
pyzotero (Python 库)
        ↑
zotero_service.py (业务封装)
        ↑
/api/zotero/* (FastAPI 路由)
        ↑
ZoteroSync 组件 (前端 React)
```

## 实施阶段

| 阶段 | 内容 | 文件数 |
|------|------|--------|
| 阶段1 | 后端：数据模型 + Zotero 服务 + API 路由 | 6 新建 + 4 修改 |
| 阶段2 | 前端：Zotero 连接配置 + 集合浏览 + 导入 UI | 2 新建 + 2 修改 |

## 关键约束

- 不新增数据库以外的持久化依赖（不引入 Redis 新用途）
- 遵循现有代码风格：服务层函数式 + Agent 类 + FastAPI 路由
- Zotero API 有速率限制，需实现缓存和退避
- API Key 敏感信息不记录到日志
