# 阶段1-01：数据模型与配置

## 1.1 Paper 模型扩展

修改 `backend/app/models/paper.py`，添加两个字段：

```python
zotero_key = Column(String(50), nullable=True, unique=True, comment="Zotero 条目 key")
zotero_synced_at = Column(DateTime, nullable=True, comment="最近一次从 Zotero 同步的时间")
```

- `zotero_key`：Zotero 中每个条目有唯一 8 位 key（如 `9KH9TNSJ`），用于去重和增量更新
- `unique=True` 防止同一 Zotero 条目被重复导入
- 两个字段均可为 NULL（非 Zotero 来源的文献留空）

## 1.2 ZoteroSync 模型

新建 `backend/app/models/zotero_sync.py`：

```python
class ZoteroSync(Base):
    __tablename__ = "zotero_syncs"

    id = Column(UUID, primary_key=True, default=uuid4)
    project_id = Column(UUID, ForeignKey("projects.id"), nullable=False)
    zotero_user_id = Column(String(20), nullable=False, comment="Zotero 用户/群组 ID")
    zotero_api_key = Column(String(64), nullable=False, comment="加密存储的 API Key")
    library_type = Column(String(10), default="user", comment="user 或 group")
    last_sync_version = Column(Integer, nullable=True, comment="上次同步的库版本号")
    synced_collections = Column(JSONB, default=list, comment="已同步的集合 key 列表")
    sync_status = Column(String(20), default="idle", comment="idle/syncing/error")
    last_sync_at = Column(DateTime, nullable=True)
    created_at = Column(DateTime, default=func.now())
    updated_at = Column(DateTime, default=func.now(), onupdate=func.now())
```

**设计说明：**
- API Key 明文存储（后续可用 Fernet 加密），当前项目处于早期开发阶段
- `last_sync_version` 支持 Zotero 增量同步（`?since=<version>` 参数）
- `synced_collections` 记录用户选择同步哪些集合
- 一个项目可以有多个 Zotero 连接（用户库 + 群组库）
