# 阶段 2 步骤 02：检索任务前端接入

## 目标

让首页检索和学术对话能展示检索任务状态，尤其是来源失败、限流、暂无相关文献等情况，避免用户看到空白结果。

## 首页接入

修改 `frontend/src/components/HomeSearchLanding.tsx`：

- 接收搜索响应中的 `task_id`。
- 在结果区域展示任务状态摘要。
- 继续展示现有来源状态条。
- 当结果为空但任务成功时，显示“暂无相关文献”。
- 当任务为 `partial` 时，显示“部分来源不可用，已展示可获取结果”。
- 当任务为 `failed` 时，展示失败来源和错误摘要。

## 学术对话接入

修改：

- `frontend/src/app/chat/page.tsx`
- `frontend/src/components/chat/ChatMessages.tsx`

目标：

- SSE `sources` 事件如果带 `task_id`，保存到 `search_results`。
- 对话消息的依据区域显示任务来源状态。
- 如果外部文献为空但内部依据存在，仍正常展示内部依据。
- 如果外部和内部都为空，显示“暂无相关文献”。

## API 与类型

修改 `frontend/src/lib/types.ts`：

```ts
export interface LiteratureSearchTaskSummary {
  id: string;
  query: string;
  mode: ResearchMode;
  library_scope: LibraryScope;
  status: "pending" | "running" | "success" | "partial" | "failed";
  total_results: number;
  source_statuses?: Record<string, SourceStatusInfo>;
  error_message?: string | null;
  created_at?: string;
  updated_at?: string;
}
```

修改 `SearchLiteratureResponse` 和 `SearchEvidenceBundle`，增加可选 `task_id`。

## 测试计划

前端构建：

```powershell
cd frontend
npm.cmd run build
```

人工验证：

- 首页搜索一个中文主题，结果区显示来源状态。
- 搜索无结果主题，显示“暂无相关文献”。
- 模拟 Semantic Scholar/OpenAlex 限流时，页面显示来源诊断。
- 学术对话开启检索后，回答下方能看到来源状态和依据区域。

## 不做内容

- 不新增后台轮询异步检索。
- 不改变首页现有视觉主结构。
