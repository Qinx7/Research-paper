# 阶段 1 步骤 01：阅读笔记最小闭环

## 目标

新增 `paper_notes` 持久化能力，让项目文献详情页可以保存阅读笔记和证据卡片。该步骤只做最小闭环：模型、Schema、API、详情页展示与编辑。

## 关键假设

- 一条笔记绑定一个 `paper_id`，可选绑定 `project_id`。
- 笔记类型先限定为：`summary`、`quote`、`method`、`finding`、`limitation`、`idea`。
- 证据可靠性先用轻量字段表达：`evidence_level`、`confidence`、`tags`。
- 先不做 PDF 坐标、划线区域、全文片段定位。

## 建议改动

### 1. 后端模型

新增 `backend/app/models/paper_note.py`：

```python
"""文献阅读笔记与证据卡片模型。"""

from sqlalchemy import Column, DateTime, ForeignKey, Integer, String, Text, func
from sqlalchemy.dialects.postgresql import JSONB, UUID
from sqlalchemy.orm import relationship

from ..core.database import Base


class PaperNote(Base):
    __tablename__ = "paper_notes"

    id = Column(UUID(as_uuid=True), primary_key=True)
    project_id = Column(UUID(as_uuid=True), ForeignKey("projects.id"), nullable=True, index=True)
    paper_id = Column(UUID(as_uuid=True), ForeignKey("papers.id"), nullable=False, index=True)
    note_type = Column(String(40), nullable=False, default="summary")
    title = Column(String(255), nullable=False)
    content = Column(Text, nullable=False)
    evidence_text = Column(Text, nullable=True)
    evidence_level = Column(String(40), nullable=True)
    confidence = Column(Integer, nullable=True)
    tags = Column(JSONB, nullable=True, default=list)
    meta = Column(JSONB, nullable=True, default=dict)
    created_at = Column(DateTime(timezone=True), server_default=func.now())
    updated_at = Column(DateTime(timezone=True), server_default=func.now(), onupdate=func.now())

    paper = relationship("Paper")
```

同步修改：

- `backend/app/models/__init__.py` 导入 `PaperNote`。
- 数据库初始化处确保模型被加载。

### 2. Schema

新增 `backend/app/schemas/paper_note.py`，包含：

- `PaperNoteCreate`
- `PaperNoteUpdate`
- `PaperNoteOut`

字段与模型保持一致，`note_type` 做基本枚举校验。

### 3. API

新增 `backend/app/api/paper_notes.py`：

- `GET /api/paper-notes?project_id=&paper_id=`
- `POST /api/paper-notes`
- `PATCH /api/paper-notes/{note_id}`
- `DELETE /api/paper-notes/{note_id}`

接入 `backend/app/main.py`：

```python
from .api.paper_notes import router as paper_notes_router

app.include_router(paper_notes_router, prefix="/api")
```

### 4. 前端类型与 API

修改 `frontend/src/lib/types.ts`：

```ts
export type PaperNoteType = "summary" | "quote" | "method" | "finding" | "limitation" | "idea";

export interface PaperNote {
  id: string;
  project_id?: string | null;
  paper_id: string;
  note_type: PaperNoteType;
  title: string;
  content: string;
  evidence_text?: string | null;
  evidence_level?: string | null;
  confidence?: number | null;
  tags?: string[];
  meta?: Record<string, unknown>;
  created_at?: string;
  updated_at?: string;
}
```

修改 `frontend/src/lib/api.ts`：

- `listPaperNotes(projectId, paperId)`
- `createPaperNote(payload)`
- `updatePaperNote(noteId, payload)`
- `deletePaperNote(noteId)`

### 5. 文献详情页接入

修改 `frontend/src/app/projects/[id]/literature/[paperId]/page.tsx`：

- 在详情页增加“阅读笔记”区域。
- 支持新建笔记，默认类型为 `summary`。
- 支持编辑标题、内容、证据摘录、标签。
- 删除前做浏览器确认。
- 空状态显示“暂无阅读笔记”。

## 风险

- 如果当前数据库没有迁移机制，新增表需要明确初始化策略。
- 如果项目尚未强制登录，需要决定笔记是否绑定用户；本步骤先绑定项目和文献。
