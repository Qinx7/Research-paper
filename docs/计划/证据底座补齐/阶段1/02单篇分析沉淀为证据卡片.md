# 阶段 1 步骤 02：单篇分析沉淀为证据卡片

## 目标

把已有单篇文献结构化分析结果转化为 `paper_notes` 证据卡片，减少分析结果只停留在页面临时状态的问题。

## 最小实现

复用现有单篇分析接口：

- `POST /api/literature/projects/{project_id}/papers/{paper_id}/analyze`

在文献详情页增加“保存为证据卡片”操作，将分析结果中的以下内容写入 `paper_notes`：

- 研究问题或核心观点：`note_type = "finding"`
- 方法信息：`note_type = "method"`
- 局限性：`note_type = "limitation"`
- 可引用证据：`evidence_text`
- 证据等级：`evidence_level`

## 后端建议

优先不改分析服务，只新增一个轻量服务函数：

`backend/app/services/paper_note_service.py`

```python
def build_notes_from_analysis(project_id, paper_id, analysis_result) -> list[dict]:
    """把单篇文献分析结果转换为可保存的证据卡片参数。"""
```

如果前端直接调用 `createPaperNote` 也能完成，则本步骤可以不新增后端服务，保持最小改动。

## 前端建议

在文献详情页的分析结果区域增加：

- “保存核心结论”
- “保存研究方法”
- “保存局限性”

保存后自动刷新阅读笔记列表。

- 对一篇文献运行结构化分析。
- 点击保存核心结论。
- 阅读笔记区域出现对应证据卡片。
- 刷新后仍能看到卡片。

## 不做内容

- 不做全文 PDF 批注。
- 不做自动生成大量笔记。
- 不强制 AI 分析一定产出结论；如果依据不足，仍显示“暂无足够依据”。
