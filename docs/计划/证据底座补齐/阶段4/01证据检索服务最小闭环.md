# 阶段 4 步骤 01：证据检索服务最小闭环

## 目标

把当前散落在聊天和论文写作中的 `PaperNote` 轻量匹配逻辑抽成一个统一服务，先做到“能稳定召回、能解释排序、能被测试覆盖”。

本步骤不引入向量检索、不新增依赖、不改变数据库结构，只基于现有 `paper_notes`、`papers` 和项目 ID 做最小闭环。

## 当前问题

- 聊天里的内部证据卡片召回逻辑写在 `backend/app/api/chat.py` 内部，后续难以复用。
- 论文写作里的证据卡片上下文写在 `backend/app/api/drafts.py` 内部，排序和过滤规则与聊天不一致。
- 当前排序只依赖简单关键词命中，缺少统一评分字段，前端也无法解释“为什么选中这条内部证据”。

## 建议改动

### 1. 新增证据检索服务

新增 `backend/app/services/evidence_retrieval_service.py`：

- `retrieve_project_evidence(db, project_id, query, limit=6, min_confidence=0)`
- `build_evidence_context(items)`
- `tokenize_evidence_query(text)`

返回统一结构：

```python
{
    "kind": "paper_note",
    "title": "...",
    "content_excerpt": "...",
    "score": 6,
    "score_reasons": ["标题命中", "证据摘录命中", "高可信度"],
    "note_type": "finding",
    "evidence_text": "...",
    "evidence_level": "摘要级证据",
    "confidence": 85,
    "source_title": "...",
    "tags": ["AI写作"],
    "action_url": "/projects/{project_id}/literature/{paper_id}",
    "action_label": "打开证据卡片",
}
```

### 2. 聊天接入

修改 `backend/app/api/chat.py`：

- `_build_project_private_context` 中不再直接查询 `PaperNote`。
- 调用 `retrieve_project_evidence(...)` 获取证据卡片。
- 仍保留项目概况、项目设计、开题报告、草稿、成果等原有上下文。
- `project_context_items` 中继续返回 `paper_note`，但增加 `score_reasons`。

### 3. 写作接入

修改 `backend/app/api/drafts.py`：

- `_build_literature_context` 调用统一服务获取高置信证据卡片。
- 写作上下文中明确提示：证据卡片只能作为已沉淀依据，不得扩写不存在的数据。
- 保留已有文献列表，不改变章节生成接口。

### 4. 前端兼容

修改 `frontend/src/lib/types.ts`：

- `ProjectContextItem` 增加 `score_reasons?: string[]`。

修改 `frontend/src/components/chat/ChatMessages.tsx` 与 `frontend/src/components/chat/ChatEvidenceRail.tsx`：

- 如果存在 `score_reasons`，以短标签显示，例如“标题命中”“高可信度”。
- 不改变聊天页主视觉结构。

## 测试计划

新增 `backend/tests/test_evidence_retrieval_service.py`：

- 查询词命中标题时，证据卡片分数更高。
- 查询词命中证据摘录时，返回 `score_reasons` 包含“证据摘录命中”。
- 低于 `min_confidence` 的证据卡片不会进入结果。
- `build_evidence_context` 输出包含“内部证据卡片”、来源文献、证据摘录和可靠性。

更新 `backend/tests/test_paper_note_evidence_reuse.py`：

- 聊天上下文仍能返回 `paper_note`。
- 写作上下文仍能包含高置信证据卡片。

运行：

```powershell
cd backend
.\.venv\Scripts\python.exe -m unittest tests.test_evidence_retrieval_service tests.test_paper_note_evidence_reuse tests.test_chat_search_threading tests.test_grounding_guard
```

前端：

```powershell
cd frontend
npm.cmd run build
```

## 验收标准

- 聊天和写作不再各自维护一套 `PaperNote` 召回逻辑。
- 内部证据卡片返回时带有 `score` 和 `score_reasons`。
- 高置信证据卡片能进入论文写作上下文。
- 前端能展示证据卡片的命中原因。
- 后端相关测试和前端构建通过。

## 不做内容

- 不新增向量化证据卡片表。
- 不改 `paper_notes` 表结构。
- 不做完整证据历史页面。
- 不重写学术对话 Agent。
