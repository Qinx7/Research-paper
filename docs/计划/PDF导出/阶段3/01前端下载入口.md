# 阶段 3 · 前端下载入口改造

## 目标

在论文草稿和开题报告下载 UI 中增加 PDF 格式选项。

## 步骤

### 3.1 API 层：下载 URL 支持 format 参数

修改 `frontend/src/lib/api.ts`，对应两处下载 URL 构造函数：

```typescript
// 论文草稿下载
export function getDraftDownloadUrl(draftId: string, format: "docx" | "pdf" = "docx"): string {
  return `${API_BASE_URL}/api/drafts/${draftId}/download?format=${format}`;
}

// 开题报告下载（如果也是 URL 构造模式）
// 若当前已是完整 URL，则追加 ?format= 参数
```

### 3.2 PaperWorkflow.tsx：下载按钮增加格式选择

在 `PaperWorkflow.tsx` 的下载按钮区域增加一个简单的格式切换：

```tsx
// 状态
const [downloadFormat, setDownloadFormat] = useState<"docx" | "pdf">("docx");

// JSX 中下载按钮旁
<div className="flex items-center gap-2">
  <select
    value={downloadFormat}
    onChange={(e) => setDownloadFormat(e.target.value as "docx" | "pdf")}
    className="text-xs border rounded px-2 py-1"
  >
    <option value="docx">DOCX</option>
    <option value="pdf">PDF</option>
  </select>
  <a href={api.getDraftDownloadUrl(draft.id, downloadFormat)} ...>
    下载论文
  </a>
</div>
```

### 3.3 PipelineWizard.tsx：开题报告下载增加格式选择

同样在 `renderStage6()` 或对应的结果展示区域，开题报告下载链接旁增加格式下拉框：

```tsx
<select
  value={downloadFormat}
  onChange={(e) => setDownloadFormat(e.target.value as "docx" | "pdf")}
  className="text-xs border rounded px-2 py-1"
>
  <option value="docx">DOCX</option>
  <option value="pdf">PDF</option>
</select>
```

### 3.4 验证

1. 进入论文阶段 → 生成草稿 → 选择 PDF 格式下载 → 文件正常打开
2. 进入开题阶段 → 生成开题报告 → 选择 PDF 格式下载 → 文件正常打开
3. 切换回 DOCX → 确认原有下载行为不受影响
