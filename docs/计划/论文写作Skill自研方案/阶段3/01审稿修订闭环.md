# 阶段3 01 审稿修订闭环

## 1. 目标

补上论文写作最关键但当前缺失的两类能力：

- 生成后的结构化审稿
- 基于问题清单的定向修订

## 2. 新增 skill

### 2.1 `paper.review_pass`

职责：

- 检查章节或全文是否存在结构问题、逻辑断裂、证据不足、表述空泛、引用缺失。

输出建议至少包括：

- `issues`
- `severity`
- `suggestions`
- `focus_areas`

### 2.2 `paper.revision_apply`

职责：

- 根据审稿问题或用户指定目标，对章节进行定向修改，而不是整章重写。

输出建议至少包括：

- `content`
- `change_summary`
- `resolved_issues`

## 3. 价值

这一步完成后，论文写作页不再只是“生成器”，而更接近真实写作工作台：

- 先写
- 再审
- 再改

## 4. 实施边界

本阶段优先做文本级修订，不做：

- track changes 富文本差异展示
- 复杂多人审阅流
- 导师协作批注
