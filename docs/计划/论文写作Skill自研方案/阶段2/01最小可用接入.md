# 阶段2 01 最小可用接入

## 1. 目标

在不破坏当前写作能力的前提下，把论文写作 skill 从“章节生成 + 章节校验”推进到“规划 + 大纲 + 章节”的最小连续链路。

## 2. 建议范围

### 2.1 新增 skill

- `paper.plan`
- `paper.outline_generate`

### 2.2 保留并兼容旧 skill

- 保留 `paper.chapter_draft`
- 保留 `paper.chapter_grounding`

本阶段先不急着改名为 `paper.section_draft / paper.citation_verify`，避免一次性影响太大。可以先在路由层支持别名，再逐步迁移。

## 3. 建议接入顺序

1. 用户先触发“生成写作计划”
2. 根据计划生成论文大纲
3. 按章节生成正文
4. 对章节做依据校验
5. 写回草稿

## 4. 用户侧效果

用户能看到的动作应是：

1. 生成写作计划
2. 生成论文结构
3. 生成本章内容
4. 校验引用依据

不应直接把内部 skill 名和执行日志暴露给用户。

## 5. 风险控制

1. 不重做现有章节 workflow。
2. 不引入额外数据库表。
3. 先用现有 `draft.outline`、`draft.content` 承接结果。
4. 若 `plan` 与 `outline` 质量不稳定，可先只做后端最小能力与接口占位，再逐步增强提示词。
