# 阶段1 01 能力模型收口

## 1. 目标

在不大改现有实现的前提下，先把论文写作 skill 的能力边界、命名和职责收口清楚。

## 2. 当前问题

当前已有的论文写作 skill 仍偏“最小实现”，主要问题有：

1. 技能数量偏少，只覆盖章节写作与依据校验。
2. 技能命名混合了“chapter”与“paper”两个层级，后续扩展会继续混乱。
3. 页面动作和 skill 动作还不是一一对应关系。
4. 缺少“规划、审稿、修订”这种真正贴近写作工作台的技能层。

## 3. 建议 skill 地图

### 3.1 写作前

- `paper.plan`
  - 输入：课题需求、项目资料、已有文献摘要
  - 输出：写作策略、建议章节顺序、风险提示、证据缺口

- `paper.outline_generate`
  - 输入：论文标题、写作策略、项目证据、目标章节集合
  - 输出：结构化大纲、每章目标、每章证据要求

### 3.2 写作中

- `paper.section_draft`
  - 输入：章节 key、章节目标、大纲、证据上下文、已有章节
  - 输出：章节正文、引用列表、数据性声明标记

- `paper.citation_verify`
  - 输入：章节结果、引用候选、内部证据、外部文献
  - 输出：校验后的章节内容、引用问题、无依据断言列表

### 3.3 写作后

- `paper.review_pass`
  - 输入：全文或章节、结构约束、写作目标
  - 输出：问题清单、修改建议、风险等级

- `paper.revision_apply`
  - 输入：原文、问题清单、修订目标
  - 输出：修订后内容、变更摘要

## 4. 与当前实现的映射

- 当前 `paper.chapter_draft` 可视为 `paper.section_draft` 的第一版。
- 当前 `paper.chapter_grounding` 可视为 `paper.citation_verify` 的第一版。
- 当前 `paper_writing_workflow` 已经具备承载连续 skill 执行的能力，不需要推翻重做。
