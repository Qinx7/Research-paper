# 阶段4 01 页面承接与产品化

## 1. 目标

把论文写作 skill 真正收敛到用户可理解的页面动作中，而不是停留在后端技术能力。

## 2. 页面层建议

### 2.1 写作页动作区

建议只保留用户能理解的动作：

1. 生成写作计划
2. 生成论文结构
3. 生成当前章节
4. 检查本章依据
5. 审查本章问题
6. 应用修订建议

### 2.2 结果展示

每个动作结果尽量展示为：

- 简洁状态
- 可读摘要
- 可执行下一步

不要展示：

- skill_id
- runtime route
- executor 细节
- 内部 workflow 运行日志

## 3. 与当前页面风格一致性

- 继续保持当前 PC 优先的写作区设计。
- 不恢复 pipeline 式开发者工作流页面。
- 让 skill 成为后端能力层，而不是前端额外概念负担。

## 4. 验收标准

如果本阶段完成，用户使用感受应该是：

- 会先得到结构建议，而不是直接整章瞎写。
- 写完后能一键检查依据问题。
- 有问题时能直接按建议修订，而不是从头重生一遍。
