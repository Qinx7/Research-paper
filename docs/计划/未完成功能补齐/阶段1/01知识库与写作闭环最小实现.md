# 阶段1 01 知识库与写作闭环最小实现

## 目标

先把“项目知识沉淀”和“论文写作状态一致性”做成一个稳定最小闭环，为后续开题主流程和交付链路打底。

## 最小范围

1. 统一草稿章节的引用、证据、真实性标记来源
2. 补一个项目级知识入口，能看到当前项目沉淀了哪些资料与证据
3. 把写作页和工作流页的公共状态规则抽成共享逻辑或共享 helper

## 本步骤不做

1. 不做导师协作
2. 不做 Elasticsearch
3. 不做大规模 UI 重构
4. 不一次性重写全部写作页面

## 交付结果

1. 用户能清楚看到项目已有的资料、证据卡片、章节引用
2. 章节编辑后状态、引用、`data_based` 规则一致
3. 后续阶段可以在统一状态模型上继续扩展
