# 阶段 3：合规检查结果可操作化

## 目标

让合规检查结果从“分数提示”升级为“可阅读、可处理的问题列表”。

## 实施内容

1. 展示合规概览：
   - 是否通过；
   - 综合分数；
   - error/warning/info 数量。

2. 展示问题列表：
   - 问题类型；
   - 严重级别；
   - 位置；
   - 描述；
   - 证据。

3. 轻操作：
   - 复制问题说明；
   - 高亮提示当前章节需要修改；
   - 后续阶段再做“确认/忽略”持久化。

## 不做内容

- 不新增后端确认接口；
- 不改合规检查算法；
- 不引入富文本定位。

## 验证方式

- 点击合规检查后，能看到具体 issues。
- 无 issues 时显示明确通过状态。
- `npm.cmd run build` 通过。
