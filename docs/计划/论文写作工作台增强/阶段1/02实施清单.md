# 阶段 1：实施清单

## 目标

本阶段只修改 `frontend/src/app/writing/page.tsx`，不改后端、不新增依赖，先解决写作页“证据状态不清楚”和“存在无效占位按钮”的问题。

## 关键假设

- 当前写作页继续使用 `textarea`，不引入富文本编辑器。
- `outcomes.length` 可作为“项目是否已有成果材料”的前端判断依据。
- `designs[0]` 可作为当前项目最近一次项目设计。
- 草稿引用来源暂时使用 `activeDraft.references`，不做后端字段扩展。
- 第 4 章和第 5 章属于结果敏感章节，需要真实成果或实验数据支撑。

## 修改范围

- 修改：`frontend/src/app/writing/page.tsx`
- 不修改：`frontend/src/lib/api.ts`
- 不修改：`frontend/src/lib/types.ts`

## 实施步骤

1. 新增章节证据状态计算。

   在已有派生状态附近增加：

   ```ts
   const hasDesign = Boolean(latestDesign);
   const hasOutcomes = outcomes.length > 0;
   const hasReferences = references.length > 0;
   const isResultSensitiveChapter = ["chapter_4_implementation", "chapter_5_experiment"].includes(activeChapterKey);
   const evidenceBadges = [
     hasDesign ? "有项目设计" : "缺项目设计",
     hasOutcomes ? "有成果材料" : "缺成果材料",
     hasReferences ? "有引用依据" : "缺引用依据",
     isResultSensitiveChapter ? "结果章节需真实数据" : null,
   ].filter(Boolean);
   ```

2. 清理顶部无效按钮。

   删除或停止渲染：

   - `FormatButton label="B"`
   - `FormatButton label="I"`
   - `FormatButton label="段落"`
   - `ToolbarButton label="改写"`
   - 不再使用的 `FormatButton` 组件
   - 不再使用的 `sidebarNoop`

   保留：

   - `续写`
   - `复制`
   - 底部 `保存`
   - 底部 `导出 DOCX`

3. 在当前章节标题区域展示证据标签。

   位置建议放在章节标题和状态行之间，避免挤占顶部工具栏。

   展示规则：

   - 有证据的标签使用主色或中性色；
   - 缺失标签使用警告色；
   - “结果章节需真实数据”使用警告色。

4. 增加结果章节风险提示。

   条件：

   ```ts
   isResultSensitiveChapter && !hasOutcomes
   ```

   提示文案：

   ```text
   当前缺少真实成果材料，本章只应生成系统设计、实验方案或待验证描述，不应生成实验结果、性能结论或具体统计数据。
   ```

5. 保持现有生成逻辑不变。

   本阶段不拦截 `handleGenerateChapter`，只在界面上明确风险。是否对生成接口增加强约束留到后续阶段处理。

## 验证步骤

1. 运行前端构建：

   ```powershell
   cd "D:\learning agent\frontend"
   npm.cmd run build
   ```

2. 人工检查 `/writing`：

   - 顶部不再出现 `B / I / 段落 / 改写` 占位按钮；
   - 第 1/2/3/6 章显示基础证据标签；
   - 切换到第 4/5 章时显示“结果章节需真实数据”；
   - 无成果材料时显示风险提示；
   - 保存、复制、续写、导出 DOCX 仍可见。

## 完成标准

- `npm.cmd run build` 通过。
- 写作页没有无效工具按钮。
- 用户能在当前章节直接看到证据状态和结果章节风险。
