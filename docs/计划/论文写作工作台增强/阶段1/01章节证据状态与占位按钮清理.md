# 阶段 1：章节证据状态与占位按钮清理

## 目标

在不改后端的前提下，让论文写作页对当前章节的证据状态更清楚，并移除当前没有实际功能的按钮。

## 当前问题

- 顶部 `B / I / 段落 / 改写` 更像视觉占位，没有真实编辑能力。
- 当前章节是否有文献、成果、项目设计支撑不够明显。
- 第 4 章“实现”和第 5 章“实验”对真实成果依赖更高，但页面没有明显风险提示。
- 用户可能误以为点击“续写”一定会生成真实结果。

## 实施内容

1. 新增章节证据状态计算：
   - 项目设计是否存在；
   - 项目成果数量；
   - 关联引用数量；
   - 当前章节是否属于结果敏感章节。

2. 顶部工具栏调整：
   - 移除 `B / I / 段落` 占位按钮；
   - 移除或禁用 `改写` 占位按钮；
   - 保留 `续写`、`复制`、`保存` 等真实可用操作。

3. 当前章节标题下增加状态标签：
   - `有项目设计`
   - `有成果材料`
   - `有引用依据`
   - `结果章节需真实数据`

4. 对第 4/5 章增强提示：
   - 若无成果材料，提示“当前只能生成设计/方案描述，不应生成实验结果或性能结论”。

## 验证方式

- 进入 `/writing` 后顶部不再出现无效格式按钮。
- 无成果项目中切到第 5 章时能看到风险提示。
- 有成果材料时提示变为“已有成果材料可供引用”。
- `npm.cmd run build` 通过。
