# 论文写作页重构推荐实施顺序

## 1. 推荐路线

我建议采用“先骨架、后联动、最后细节”的顺序，而不是一开始就补很多按钮或格式能力。

原因很直接：

- 当前最大问题是页面角色错了，不是控件少了；
- 如果先补功能，容易继续把页面做成复杂控制台；
- 先把正文区、章节导航、依据面板三者关系理顺，后续再补细节才不会返工。

## 2. 分步建议

### 第一步：布局骨架重构

目标：

- 去掉明显的工作流感；
- 正文区成为主要视觉中心；
- 右侧面板改为可开合或弱化存在。

落点：

- `/writing` 主体三栏关系重排；
- 顶部状态精简；
- 底部状态栏收敛。

### 第二步：正文区写作感重构

目标：

- 让页面更像正式文稿编辑器；
- 明确当前章节、当前文稿、当前写作状态；
- 提升长时间写作舒适度。

落点：

- 标题区层次；
- 段落留白；
- 文稿内状态展示；
- 章节切换的连续感。

### 第三步：依据联动重构

目标：

- 让依据服务当前章节，而不是服务“整个项目概览”；
- 降低来回翻找证据的成本。

落点：

- 当前章节依据摘要前置；
- 文献、成果、资料片段分组但不泛滥；
- 右栏改成“当前章补充信息”而不是“全局资料仓”。

### 第四步：交付与次级功能收口

目标：

- 保留导出等必要能力；
- 把答辩和交付类内容从主写作心智中弱化。

落点：

- 只保留必要导出；
- 交付状态不再长期占据主面板；
- 写作页聚焦正文生产。

## 3. 我建议优先做的版本

如果只做一轮最有效的改动，我建议先做：

1. 布局骨架重构
2. 正文区写作感重构
3. 当前章节依据前置

这三项做完，用户主观感受会明显变化，且不需要立即引入新的编辑器依赖或重写后端。

## 4. 暂不建议优先做的内容

- 真富文本编辑器接入
- 复杂格式工具条
- 批注/修订模式
- 真分页打印排版

这些都属于高成本项，应该在写作主体验已经成立之后再决定。
