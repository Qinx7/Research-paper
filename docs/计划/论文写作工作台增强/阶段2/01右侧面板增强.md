# 阶段 2：右侧面板增强

## 目标

增强右侧 `建议 / 引用 / 交付` 面板，让写作过程能围绕证据和交付状态推进。

## 实施内容

### 建议面板

- 根据当前章节类型展示不同建议：
  - 第 1/2 章：强调文献依据和研究背景；
  - 第 3 章：强调项目设计、方法和技术路线；
  - 第 4/5 章：强调真实成果、实验数据和截图；
  - 第 6 章：强调总结与限制，不新增未验证结论。

### 引用面板

- 分组展示：
  - 项目设计依据；
  - 草稿引用；
  - 项目成果材料；
  - 空状态提示。

### 交付面板

- 显示：
  - 草稿进度；
  - 当前章节状态；
  - 成果材料数量；
  - 答辩 PPT 大纲状态；
  - DOCX/PDF 下载入口。

## 验证方式

- 切换章节时建议文案会随章节变化。
- 引用面板不再把所有引用混成一列 JSON 字符串。
- 交付面板能看到导出和答辩相关状态。
- `npm.cmd run build` 通过。
