# 阶段 2：研究方向视图

## 目标

补齐截图 2 对应的研究方向页面视觉结构，让当前项目不再只有开题流程里的普通方向卡片。

## 最小实现

- 读取当前 `PipelineWizard` 第 4 步已有的 `directions` 和 `scores` 数据。
- 将研究方向展示改成左侧方向列表 + 右侧详情分析的布局。
- 对真实数据缺失的字段做保守映射，不编造论文趋势、顶会、学者等事实数据。
- 对无数据字段显示“暂无数据”或基于已有字段的简单展示。

## 验证方式

- 第 4 步生成研究方向后，页面布局接近截图 2。
- 选择左侧方向后，右侧详情跟随更新。
- 不改变 `generateDirections` 和 `generateDesign` 的 API 调用。
- 运行 `npm.cmd run build`。

## 风险

- 参考设计里的趋势图、代表学者、顶会数据是静态样例，当前后端未必返回这些字段。实现时必须避免把样例数据当作真实结果。
