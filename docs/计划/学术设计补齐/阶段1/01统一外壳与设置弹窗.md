# 阶段 1：统一外壳与设置弹窗

## 目标

先补齐最小工作台外壳：让侧栏不再只是聊天历史侧栏，而能表达参考设计里的三大模块，并新增设置弹窗入口。

## 最小实现

- 保留 `/chat` 当前学术搜索页面。
- 在聊天侧栏底部用户区增加打开设置弹窗的交互。
- 新增或抽取一个 `SettingsPanel` 聊天/工作台可复用组件。
- 设置弹窗先实现个人信息页，可预留研究偏好、AI 设置、通知、隐私安全导航，但不接入真实保存 API。

## 验证方式

- `/chat` 可以打开和关闭设置弹窗。
- 设置弹窗布局与截图 3 接近。
- 不影响现有聊天发送、历史选择和删除。
- 运行 `npm.cmd run build`。

## 风险

- 当前项目没有统一多视图工作台路由，如果一次性把研究方向、论文写作都塞进 `/chat`，可能与首页/项目详情页职责冲突。因此阶段 1 只做外壳和设置弹窗，不重排业务路由。
