# 学术设计补齐总纲

## 需求理解

用户提供的本地下载设计项目不是单一“文献搜索页”，而是一个完整的学术工作台原型，包含：

- 文献搜索页：`SearchView.tsx`
- 研究方向页：`ResearchView.tsx`
- 论文写作页：`WritingView.tsx`
- 设置弹窗：`SettingsPanel.tsx`
- 统一侧栏与应用容器：`Sidebar.tsx`、`App.tsx`

上一轮实现主要覆盖了文献搜索页对应的聊天页外观，没有完整覆盖研究方向、论文写作和设置弹窗，因此不能宣称已经完整读取和实现下载项目。

## 当前证据

- 本地设计项目的 `App.tsx` 明确维护 `activeView: "search" | "research" | "writing"`，说明参考设计是三视图工作台。
- 截图 1 对应论文写作页：左侧主侧栏 + 论文结构栏 + 中央编辑器 + 右侧建议/引用栏。
- 截图 2 对应研究方向页：左侧方向列表 + 右侧统计、趋势、顶会、细分方向、学者、研究空白。
- 截图 3 对应设置弹窗：居中弹窗 + 左侧设置导航 + 右侧表单。
- 当前项目只有 `/chat`、`/projects/[id]`、首页开题流程等路由；研究方向能力主要在 `PipelineWizard` 第 4 步，论文写作能力主要在 `PaperWorkflow`，设置弹窗尚无对应完整实现。

## 关键假设

- 目标是让当前项目的相关页面整体接近本地设计项目，而不是只让 `/chat` 接近 `SearchView`。
- 首页已有单独安排，本轮仍不主动重做首页。
- 不新增依赖，继续使用 React、Next.js、Tailwind 和现有业务 API。
- 优先保持现有后端数据流和功能可用，UI 对齐以组件结构和视觉节奏为主。

## 实施范围

本方案建议覆盖以下最小必要范围：

- `/chat`：保留已完成的文献搜索工作台样式，并补通侧栏的视图切换入口。
- 研究方向视图：基于 `ResearchView.tsx` 的布局，在当前项目中形成可访问的研究方向页或视图。
- 论文写作视图：基于 `WritingView.tsx` 的布局，改造 `PaperWorkflow` 中的论文写作阶段，使其更像编辑器而不是流程卡片。
- 设置弹窗：新增与参考 `SettingsPanel.tsx` 对齐的个人设置弹窗，至少可由侧栏底部用户区打开。

## 建议阶段

1. 阶段 1：补齐统一工作台外壳与设置弹窗。
2. 阶段 2：补齐研究方向页视觉结构。
3. 阶段 3：补齐论文写作编辑器视觉结构。

## 验收标准

- 能从侧栏看到文献搜索、研究方向、论文写作三个模块入口。
- 设置弹窗与截图 3 的布局相似：左侧头像/导航，右侧表单，居中遮罩弹窗。
- 研究方向页与截图 2 的布局相似：左列表、右详情、统计卡、趋势图、顶会、方向标签、学者、研究空白。
- 论文写作页与截图 1 的布局相似：左结构栏、中央大编辑区、右建议/引用栏。
- `npm.cmd run build` 通过。

## 需要确认

请确认是否开始按该方案实施。建议先执行阶段 1，因为它能最小化地修正“参考项目是多视图工作台”的结构遗漏，并为后续研究方向页和写作页铺好入口。
