# 阶段2：写作到项目页跳转定位增强

## 1. 目标

让用户从写作页点击依据后，尽量落到项目页中的准确位置，而不是只到一个泛入口。

## 2. 最小实现

### 2.1 跳转参数统一

统一使用清晰参数表达：

1. `view`
2. `tab`
3. `highlight`
4. `item_id`

以便项目页接收后做自动展开或高亮。

### 2.2 写作页动作统一输出

依据动作生成时尽量带完整上下文：

- 目标对象类型
- 对象 id
- 所属模块
- 是否需要高亮

### 2.3 项目页自动定位

项目页收到参数后：

1. 自动切到对应模块；
2. 自动展开对应列表；
3. 高亮命中的对象；
4. 用户一眼知道“这就是写作页里那条依据”。

## 3. 验收

- 从写作页点成果，能看到准确成果项；
- 点资料片段，能看到对应资料位置；
- 点文献，能到项目文献库对应项；
- 点证据，能看到对应证据卡片。
