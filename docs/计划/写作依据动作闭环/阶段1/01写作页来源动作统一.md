# 阶段1：写作页来源动作统一

## 1. 目标

解决当前写作页里“依据都能点，但动作语义不统一”的问题。

## 2. 当前问题

- 成果有时是下载，有时是查看；
- 资料片段、文献、证据卡片的文案不一致；
- 用户点开前不清楚会跳到项目页、资料页还是下载文件；
- 同一类对象在不同页里的动作名字可能不同。

## 3. 最小实现

### 3.1 统一动作文案

建议统一为：

1. 成果：`查看成果`
2. 资料片段：`查看片段`
3. 文献：`查看文献`
4. 证据：`查看证据`

如果确实是文件下载，再附加下载动作，而不是只给下载。

### 3.2 补来源说明

在每条依据卡片中，增加一句更短的来源提示，例如：

- 来自项目成果
- 来自上传资料的章节片段
- 来自项目文献库
- 来自证据卡片

## 4. 验收

- 同类依据动作命名一致；
- 用户不需要猜“点一下会去哪里”；
- 写作页观感仍保持简洁。
