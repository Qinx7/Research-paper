# 文献阅读与证据沉淀总纲

## 需求理解

在已经完成“搜索结果加入项目文献库”的基础上，下一步需要让用户能够围绕单篇文献进行阅读、核验和结构化沉淀。目标不是马上做完整 PDF 阅读器，而是先把“项目文献库 → 文献详情 → 结构化理解 → 后续综述/矩阵”这条最短路径跑通。

## 关键假设

- 先复用现有 `papers` 表，不新增数据库表。
- 第一版文献详情以元数据、摘要、来源链接和 AI 结构化分析为主，不处理 PDF 高亮、段落定位和划线笔记。
- AI 分析必须基于已保存的题名、摘要、作者、期刊、年份、来源等字段；缺少摘要时需要明确提示“证据不足”，避免凭空生成。
- 文献详情页作为项目详情页的子能力，从“项目文献库”进入。

## 阶段划分

### 阶段1：文献详情页最小实现

- 在项目文献库卡片中增加“查看详情”入口。
- 新增文献详情展示能力。
- 展示文献标题、作者、年份、来源、DOI、摘要、引用量、原文链接。
- 对缺失字段给出明确空状态。

### 阶段2：单篇文献结构化分析

- 增加后端接口：输入项目 ID 与文献 ID，返回结构化分析结果。
- 分析字段包括研究问题、研究方法、研究对象/样本、主要发现、局限、与当前项目的相关性。
- 如果摘要不足，返回“暂无足够依据”，不编造方法、样本或结论。

### 阶段3：文献矩阵

- 基于项目文献库生成矩阵视图。
- 字段包括作者年份、研究主题、方法、样本、结论、局限、可借鉴点。
- 支持复制或导出为 Markdown，方便后续写综述。

## 当前优先级

优先执行阶段1和阶段2的最小版本。阶段3等单篇分析稳定后再做。

## 验收标准

- 用户能从项目文献库进入某篇文献详情。
- 文献详情页能展示完整元数据和摘要。
- 单篇 AI 分析不脱离已有字段；摘要缺失时不编造。
- 分析结果能作为后续文献矩阵的基础。
