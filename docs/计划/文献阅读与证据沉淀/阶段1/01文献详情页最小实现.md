# 阶段1 01：文献详情页最小实现

## 目标

给项目文献库增加单篇文献详情视图，让用户能够从收藏列表进入一篇文献，查看完整元数据和摘要。

## 修改范围

- 前端项目详情页或文献库组件。
- 前端 API 封装必要补充。
- 后端可优先复用现有项目文献列表接口；如果需要单篇详情，再补 `GET /api/literature/projects/{project_id}/papers/{paper_id}`。

## 页面内容

- 标题
- 作者
- 年份
- 来源数据库
- 期刊/会议
- DOI
- 引用量
- 摘要
- 原文链接
- 返回项目文献库

## 空状态规则

- 没有摘要：显示“当前文献暂无摘要，建议打开来源链接核验”。
- 没有 DOI：显示“暂无 DOI”。
- 没有来源链接：隐藏打开来源按钮。

## 验证方式

- 从项目文献库点击“查看详情”能打开详情。
- 刷新详情页后仍能通过项目 ID 和文献 ID 加载。
- 前端构建通过。
