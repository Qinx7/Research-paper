# P2 中长期能力缺口

## 1. 独立全文检索引擎

### 当前状态

- 当前项目资料搜索已经能用
- 但实现仍是数据库读取 + 关键词打分
- 没有 `Elasticsearch / OpenSearch` 这类独立全文检索引擎

### 为什么是 P2

当前规模下未必是最紧迫问题。只要项目资料语义检索闭环先做好，独立全文检索引擎可以后置。

## 2. 更完整的权威目录运营能力

### 当前状态

- 已经预留本地目录匹配服务
- 但没有看到权威目录的版本管理、更新策略、校验规则运维能力

### 为什么是 P2

先把“能核验”做出来更重要；目录运营能力属于后续放大阶段。

## 3. 更深的论文级结构化解析栈

### 当前状态

- 已有文本提取、OCR、章节切分、部分结构化结果展示
- 但没有完整的 `GROBID + 论文元数据深抽取 + 参考文献结构化校对` 产品栈

### 为什么是 P2

如果当前目标是先稳住主链路，这一层更适合在知识底座和语义检索闭环稳定后再做。

## 4. 建议不纳入当前版本主目标的事项

以下项可以继续保留在长期路线中，但不建议抢占当前阶段资源：

1. 大规模团队协作产品化
2. 独立全文检索集群化部署
3. 完整论文出版级引用网络分析
4. 完整外部目录运营后台
