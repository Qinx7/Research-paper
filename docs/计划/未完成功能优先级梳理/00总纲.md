# 未完成功能优先级梳理

## 1. 目标

基于当前 `literature_driven_research_agent_prd.md`、`docs/产品/PRD_v2_产品收敛版.md` 与现有代码实现，整理出当前版本仍未真正完成的功能清单，并按 `P0 / P1 / P2` 划分优先级。

本清单只统计“当前产品边界内仍应实现，但尚未完整落地”的事项：

- 不再把已下线的 `proposal / 开题报告 / 答辩主链路` 记为未实现
- 不再把“已有入口但还能继续打磨”的页面体验误记成“功能未实现”
- 优先关注会直接影响当前主链路可信度、可用性和闭环能力的缺口

## 2. 当前主链路基线

当前版本主链路已经收敛为：

1. 首页学术检索
2. 需求理解 / 文献分析 / 研究方向生成
3. 研究页承接研究方向与项目设计
4. 项目页沉淀文献、成果、证据卡片、资料片段
5. 写作页完成整篇论文生成、编辑、审查、修订、导出
6. 基于项目设计或论文草稿生成通用 PPT / HTML Deck

## 3. 优先级判断原则

### P0

直接影响当前主链路是否可信、是否可持续复用、是否能形成真正闭环。

### P1

不阻塞当前主链路，但会明显影响专业度、搜索质量或产品上限。

### P2

有价值，但不应抢占当前主链路稳定化资源。

## 4. 结论总览

### P0

1. 权威目录级核验能力
2. 项目私有资料语义检索闭环
3. 文献全文解析产品化补齐

### P1

1. 多 Agent 编排框架正式化
2. 严格引用网络级文献知识图谱
3. 导师 / 团队协作与权限分层

### P2

1. 独立全文检索引擎
2. 更完整的权威目录运营能力
3. 更深的论文级结构化解析栈
