# P1 能力增强缺口

## 1. 多 Agent 编排框架正式化

### 当前状态

- 已有自研 `workflow + skill runtime`
- 首页检索、研究方向、写作等路径已经开始接入
- 但这还不是原设想中的标准化多 Agent 编排框架

### 为什么是 P1

它不阻塞当前主链路能跑，但会影响后续复杂链路扩展、节点诊断、可复用性和长期维护成本。

### 涉及页面

- 首页搜索分析链
- 研究页方向/设计链
- 写作页整篇生成/审查/修订链

### 涉及后端模块

- `backend/app/agents/workflows/*`
- `backend/app/skills/*`
- `backend/app/services/agent_workflow_record_service.py`

## 2. 严格引用网络级文献知识图谱

### 当前状态

- 项目中已有知识图谱可视化
- 但关系更多来自关键词、作者、年份、聚类
- 不是严格的引用网络、共引网络、共被引网络

### 为什么是 P1

它会提升专业度，但不阻塞当前检索、研究、写作主链路。

### 涉及页面

- 项目页 `知识图谱`
- 文献矩阵 / 文献详情联动

### 涉及后端模块

- `backend/app/api/literature*.py`
- `backend/app/services/*knowledge_graph*`

## 3. 导师 / 团队协作与权限分层

### 当前状态

- 当前系统仍是单用户项目模式
- 没有看到完整的导师审阅、批注、共享项目、团队角色模型

### 为什么是 P1

这是明显的产品扩展项，但不属于当前单用户主链路的最短板。

### 涉及页面

- 项目页
- 写作页
- 用户中心 / 权限管理

### 涉及后端模块

- `backend/app/models/*user*`
- 新的 team / role / share / review 模块
