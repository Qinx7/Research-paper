# 阶段1：结果卡片解释收口

## 1. 目标

让用户在单篇文献卡片上直接看懂：

1. 为什么命中
2. 为什么推荐
3. 当前标签代表什么

## 2. 当前问题

- `why_selected` 有时存在，但展示不够突出；
- `authority_reasons` 位置偏散，不够统一；
- 用户容易把“权威标签”理解成“绝对高质量”；
- 相关性、来源、方法匹配等原因没有明确分层。

## 3. 最小实现建议

每篇结果卡片统一成 3 段：

1. 主信息
   - 标题
   - 作者 / 年份 / 来源
   - 标签

2. 为什么命中
   - 显示 `why_selected`
   - 没有时回退摘要首句或通用提示

3. 来源说明
   - 显示 `authority_reasons`
   - 显示待核验提示

## 4. 不做事项

- 不做复杂可折叠打分面板
- 不新增大型交互组件
- 不一次性重写整页布局

## 5. 验收

- 每张卡片都有稳定解释区
- 解释区文案位置统一
- “已核验 / 待核验 / 推断”语义不混乱
