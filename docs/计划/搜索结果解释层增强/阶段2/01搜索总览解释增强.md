# 阶段2：搜索总览解释增强

## 1. 目标

让用户一眼看懂“这次搜索整体是否可靠、哪些来源异常、结果是否完整”。

## 2. 当前问题

- `authority_summary` 和来源状态存在，但信息组织不够直观；
- 用户不容易分辨“来源没查到”和“来源查失败”；
- 当前总览对这次搜索的质量判断不够显式。

## 3. 最小实现建议

把总览解释拆成三块：

1. 结果可信度概览
   - 已核验文献数量
   - 待核验文献数量
   - 搜索整体说明

2. 来源状态
   - 成功返回
   - 空结果
   - 限流 / 超时 / 被拦截 / 失败

3. 风险提示
   - 当前是否存在关键来源缺口
   - 当前结论是否应谨慎参考

## 4. 验收

- 用户不再把“空结果”和“来源失败”混为一谈
- 总览区能直接说明这次检索的质量边界
