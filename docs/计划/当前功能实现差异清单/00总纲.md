# 当前功能实现差异清单

## 1. 目的

本清单用于对照以下三类文档，统一梳理当前项目“已经完成了什么、还差什么、哪些只是半完成、哪些暂时不纳入当前版本”：

1. 原始需求文档 [literature_driven_research_agent_prd.md](/D:/learning%20agent/literature_driven_research_agent_prd.md)
2. 当前产品基线 [docs/产品/PRD_v2_产品收敛版.md](/D:/learning%20agent/docs/产品/PRD_v2_产品收敛版.md)
3. 对外说明 [README.md](/D:/learning%20agent/README.md)

## 2. 评估口径

### 2.1 已完成

满足以下条件：

1. 用户侧有明确入口
2. 前后端都已打通
3. 可真实使用，不依赖手工绕路

### 2.2 半完成

满足以下特征之一：

1. 后端能力存在，但前端入口不完整
2. 页面可见，但只做到了展示，没有动作闭环
3. 功能能跑，但和文档中的承诺相比仍缺关键环节

### 2.3 未完成

满足以下特征之一：

1. 只有文档，没有实际产品入口
2. 只有局部代码雏形，没有稳定链路
3. 原始需求文档中明确提到，但当前代码未真实落地

### 2.4 暂不纳入当前版本

这些能力可能在原始文档中有，但已被 `PRD_v2` 明确收缩出当前产品边界，不作为当前版本交付目标。

## 3. 总体判断

### 3.1 按 PRD_v2 看

当前项目已经基本完成：

1. 首页学术检索主链路
2. 三步分析
3. 项目知识底座
4. 论文写作基础链路
5. 研究方向到项目设计、开题材料的承接主链路

但仍有若干“收口型缺口”没有完全完成。

### 3.2 按原始需求文档看

当前项目还没有完全达到“科研全过程平台”的完整实现状态。

尤其在以下方面仍有明显差距：

1. 官方级权威目录核验
2. IEEE / ACM 文献识别策略与来源说明仍需进一步收口
3. 团队协作与导师审阅
4. 中期汇报与阶段性材料链路
5. 全文检索引擎
6. 正式迁移体系
7. 更完整的答辩交付链路

## 4. 推荐用法

本清单建议作为后续迭代优先级判断依据：

1. 优先补“半完成但直接影响主链路体验”的功能
2. 对“未完成但仍在 PRD_v2 边界内”的能力，作为下一阶段开发目标
3. 对“暂不纳入当前版本”的能力，不继续分散资源
