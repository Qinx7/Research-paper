# 半完成功能

## 1. 写作依据动作闭环

当前状态：

1. 已能看到“当前章节用了哪些成果 / 资料片段”
2. 项目页也能看到“某个成果被哪些章节使用”

仍缺：

1. 写作页中这些依据还不是完整可点击跳转闭环
2. 成果、资料片段、文献、证据卡片之间的来源动作不统一
3. 用户还不能从每一条依据直接打开最准确的来源页面

## 2. 交付工作台

当前状态：

1. 项目页已有交付工作台
2. 可看到草稿、开题报告、答辩大纲等状态

仍缺：

1. 毕业答辩 PPT 文件与统一下载区没有完全产品化
2. 交付材料归档还不完整
3. 答辩材料说明与统一交付动作还不完整

## 3. 研究页连续承接

当前状态：

1. 研究页本身已产品化
2. 方向 → 设计 → 开题材料承接成立

仍缺：

1. `需求理解 → 文献分析 → 研究方向 → 研究页` 仍跨页面完成
2. 还不是单页完整连续入口

## 4. 学术搜索质量增强

当前状态：

1. 已增强权威标签
2. 已增强来源诊断
3. 已有多来源聚合

仍缺：

1. 搜索结果“为什么命中”解释还不够系统
2. 来源质量解释面板还不够完整
3. 真实来源能力和推断标签能力仍混在一部分摘要语义里

## 5. 项目成果到写作的闭环提示

当前状态：

1. 成果已能入知识库
2. 写作页开始能看到当前章节命中的资料

仍缺：

1. 对“为什么命中本章”的解释还不够强
2. 成果命名和章节命中的鲁棒性仍依赖文本匹配
3. 没有更强的“引用来源确认”动作
