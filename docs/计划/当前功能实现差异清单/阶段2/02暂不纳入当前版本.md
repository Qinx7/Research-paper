# 暂不纳入当前版本

这些能力在原始需求文档中出现过，但已经被 `PRD_v2` 主动收缩出当前版本目标。

## 1. 导师批注与审阅模式

原因：

1. 当前产品优先做学生自助主链路
2. 导师场景涉及权限、协作、流程管理，超出当前收敛目标

## 2. 团队共享与权限系统

原因：

1. 当前版本不是团队协同平台
2. 权限和共享会显著扩大系统复杂度

## 3. 通用开发者执行记录工作台

原因：

1. 对用户价值弱
2. 更偏内部观测和排障能力

## 4. 通用多 Agent 可视化编排平台

原因：

1. 当前项目更需要稳定主链路
2. 可视化编排平台属于中长期能力

## 5. 全文检索平台级建设

原因：

1. 当前已有项目知识库和向量检索可以支撑主链路
2. 搜索平台级基础设施不属于当前版本优先级

## 6. 覆盖科研全过程的一体化万能工作台

原因：

1. `PRD_v2` 已明确首页和各页面职责收窄
2. 当前策略是聚焦主闭环，而不是继续堆大全功能平台
