# 研究方向页 Stitch 布局改造总纲

## 需求理解

参考 Stitch 页面 `Research Direction Analyzer / 文献研究方向分析 - 视觉升级版`，将当前 `/research` 页面改造成更像“研究方向分析仪表盘”的页面。

Stitch 页面核心结构是：

- 顶部固定导航栏：品牌、页面入口、全局搜索、通知、设置、头像。
- 左侧分析导航栏：项目卡片、分析维度导航、导出报告入口。
- 主内容区：研究主题标题、检索关键词、筛选标签。
- 分析卡片区：研究趋势图、领域分布、关键词云、领军研究人员。
- 文献卡片区：高影响力论文列表。

当前项目 `/research` 页面已有真实功能：

- 读取项目列表，选择当前项目。
- 读取候选研究方向列表。
- 搜索候选方向。
- 预览候选方向详情。
- 设置当前研究方向。
- 展示方向评分。
- 基于研究方向生成项目设计。
- 基于当前项目设计生成通用 PPT。
- 进入论文写作页继续承接。

本次目标不是新增后端能力，而是用 Stitch 的布局与视觉语言重组这些既有功能。

## 关键假设

- 本轮优先只修改 `frontend/src/app/research/page.tsx`。
- 不新增依赖、不改后端接口、不改数据库。
- Stitch 中的假数据模块必须映射为我们已有数据，不能凭空伪造“高引用论文、作者 H 指数、年度真实趋势”等不可验证内容。
- 如果某些分析模块缺少真实数据，则以“候选方向评分、方向数量、研究问题/方法/创新点/风险”等现有字段生成可解释的前端可视化。
- 页面整体保持浅灰蓝、白色卡片、深色主按钮的专业学术仪表盘风格，与首页和写作页保持同一套克制颜色。

## 功能映射

### 顶部导航

- Stitch `ScholarCanvas` 映射为“研究方向分析”。
- `Dashboard / Library / Datasets` 映射为：首页、学术对话、论文写作。
- 顶部搜索框映射为当前 `query`，用于搜索候选研究方向。
- 头像区域使用登录用户首字母兜底。

### 左侧导航

- 项目卡片映射为当前 `selectedProject`。
- `Trends` 映射为“方向总览”。
- `Citations` 映射为“研究问题”。
- `Methodology` 映射为“方法路径”。
- `Authors` 映射为“创新与风险”。
- `Funding` 映射为“项目承接”。
- `EXPORT REPORT` 映射为当前通用 PPT 生成或下载状态。

### 主趋势图

Stitch 的年度趋势图不直接使用虚构年份数据。改为展示候选方向的评分分布：

- 每个候选方向一根柱。
- 高度来自 `overall` 或 `recommendation_score`。
- 当前预览方向高亮。
- 当前已确认方向显示特殊标记。

### 指标卡片

Stitch 的三张指标卡映射为：

- 研究方向评分雷达/环形卡：展示文献基础、创新性、可行性、数据可得性、论文价值。
- 核心关键词/要点云：从研究问题、方法、创新点、预期产出中提取短词或短句。
- 承接状态卡：显示当前方向、项目设计、PPT、论文写作入口。

### 高影响力论文区

当前 `/research` 页面没有直接拿到文献卡片数据，因此不伪造论文列表。本区改为“候选研究方向列表”或“研究问题与方法证据卡”：

- 左侧显示推荐等级、方向评分、是否当前方向。
- 中间显示方向标题、背景摘要、研究问题。
- 右侧提供“预览”“设为当前方向”“生成项目设计”等动作。

## 成功标准

- 页面视觉上接近 Stitch：顶部栏、左侧分析导航、主趋势分析卡、三张指标卡、下方方向卡片列表。
- 当前已有功能不丢失：项目选择、方向搜索、方向预览、设为当前方向、生成项目设计、生成 PPT、进入论文写作。
- 不显示不可验证的假论文、假引用数、假作者指标。
- 无项目、无方向、未登录、加载中状态仍有清晰提示。
- 页面主体内部滚动，避免整个页面布局松散或过长。
- `npm.cmd run build` 通过。

## 不纳入本轮

- 不新增真实文献趋势后端接口。
- 不新增作者网络、基金分析、年度论文数统计。
- 不新增图表库，趋势图和环形评分优先用 CSS/SVG 轻量实现。
- 不重构项目设计/PPT 生成接口。
