# 阶段3 01 构建与交互验证

## 目标

完成页面构建验证和关键交互验证，确保研究方向页视觉改造后仍能承接后续流程。

## 验证清单

- `npm.cmd run build` 通过。
- 未登录状态显示登录提示。
- 无项目状态显示返回首页入口。
- 有项目但无研究方向时显示清晰空状态。
- 顶部搜索能过滤候选方向。
- 点击左侧方向或趋势柱能切换预览方向。
- 点击“设为当前方向”能调用原 `handleSetCurrentDirection`。
- 点击“生成项目设计”能调用原 `handleGenerateDesign`。
- 点击“生成 PPT”能调用原 `handleGenerateProjectPpt`。
- 点击“进入论文写作”能带 `project_id` 跳转到 `/writing`。

## 适配要求

- 桌面端优先，保持顶部栏 + 左侧栏 + 主内容区。
- 中等宽度下允许主内容滚动，不让卡片挤压变形。
- 小屏可以退化为上下布局，但功能入口不能消失。

## 交付说明

完成后需要说明：

- 研究方向页改成了哪些区域。
- Stitch 中哪些模块被替换为我们的真实功能。
- 哪些假数据没有照搬。
- 构建验证结果。
