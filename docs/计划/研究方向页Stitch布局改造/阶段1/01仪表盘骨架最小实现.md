# 阶段1 01 仪表盘骨架最小实现

## 目标

先用最小改动把 `/research` 页面改成 Stitch 风格的研究方向分析仪表盘骨架，保留现有数据和处理函数。

## 范围

- 修改 `frontend/src/app/research/page.tsx` 的页面结构和样式。
- 保留当前 state、effect、handler 和 API 调用。
- 不引入新组件库，不新增依赖。

## 具体步骤

1. 将页面外壳改为固定高度的顶部栏 + 左侧栏 + 主内容区。
2. 顶部栏放品牌、入口跳转、方向搜索、用户状态。
3. 左侧栏放项目选择、分析维度导航、方向列表摘要、PPT/报告入口。
4. 主内容区顶部放当前方向标题、项目字段、状态标签。
5. 主内容区第一张大卡放“候选方向评分趋势图”。
6. 下方放三张指标卡：方向评分、关键词/要点云、承接状态。
7. 最下方放候选方向卡片列表，承载预览与动作入口。

## 验证

- 登录后能加载项目与方向。
- 顶部搜索仍过滤候选方向。
- 选择候选方向后主内容随之切换。
- 设为当前方向、生成项目设计、生成 PPT、进入论文写作按钮仍连接原函数。
- `npm.cmd run build` 通过。

## 风险

- 当前研究页已有较多状态分支，重排 JSX 时容易漏掉无项目/无方向/加载中分支。
- 需要避免把 Stitch 的静态假数据写进页面，所有展示应来自当前项目和候选方向。
