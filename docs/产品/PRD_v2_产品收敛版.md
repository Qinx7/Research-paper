# 文献驱动科研助手 PRD v2

## 1. 文档目的

本版本作为当前产品的实施基线，描述已经收敛后的真实产品边界。

## 2. 一句话定位

这是一个面向研究生科研场景的文献检索、选题分析、项目设计、知识沉淀、论文写作与通用 PPT 展示产品。

## 3. 当前主链路

```text
首页检索
→ 需求理解
→ 文献分析
→ 研究方向生成
→ 项目设计
→ 成果与资料入知识库
→ 论文写作
→ 通用 PPT / HTML Deck
```

## 4. 当前页面职责

### 4.1 首页

- 学术检索入口
- 检索结果展示
- 本次检索的需求理解、文献分析、研究方向结果
- 检索记录恢复

### 4.2 研究页

- 研究方向承接
- 项目设计生成
- 通用 PPT 生成入口

### 4.3 项目页

- 项目概览
- 项目知识工作台
- 文献库与文献矩阵
- 论文工作流入口
- HTML Deck 预览入口

### 4.4 写作页

- 草稿创建
- 章节生成与连续编辑
- 依据追踪
- 合规检查
- 导出

## 5. 当前版本保留能力

1. 多来源学术检索
2. 检索解释层与来源状态展示
3. 研究方向生成与评分
4. 项目设计生成
5. 成果文件解析入知识库
6. 文献笔记 / 证据卡片 / 资料片段沉淀
7. 论文写作与导出
8. 通用 PPT / HTML Deck 生成

## 6. 当前版本明确下线能力

以下能力已不属于当前产品范围：

1. 开题报告生成
2. 开题报告下载
3. `proposal` 模型与接口
4. 基于开题报告的 PPT 生成
5. 基于 `proposal_id` 的 HTML Deck 预览

## 7. 当前接口边界

### 保留

- `/api/literature/*`
- `/api/research/*`
- `/api/drafts/*`
- `/api/ppt/styles`
- `/api/ppt/generate`
- `/api/ppt/html-deck`

### 移除

- `/api/proposal/*`
- `/api/ppt/proposal`

## 8. P0 验收标准

1. 首页历史搜索可恢复且不重复生成
2. 研究方向与项目设计可承接、可保存
3. 上传资料可进入项目知识库
4. 写作页可稳定生成、编辑、导出论文草稿
5. 通用 PPT / HTML Deck 可从项目设计或论文草稿生成

## 9. 技术约束

1. 优先保证主链路稳定
2. 使用 Alembic 维护数据库迁移
3. 不再为已下线的开题报告能力保留产品承诺
4. 当前版本主打 PC 端体验
