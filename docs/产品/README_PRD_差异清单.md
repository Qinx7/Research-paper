# README / 原 PRD 与 PRD v2 差异清单

## 1. 文档角色变化

### 原 README / 原 PRD

定位更接近：

1. 愿景说明
2. 大范围功能蓝图
3. 长期系统能力清单

### PRD v2

定位更接近：

1. 当前产品实施基线
2. 页面边界说明
3. 当前版本优先级与范围约束

## 2. 核心定位差异

### 原文档倾向

强调“覆盖研究生科研全过程的智能平台”，包括：

1. 选题
2. 开题
3. 项目设计
4. 论文
5. 协作
6. 团队管理

### v2 倾向

强调“先做稳定主链路”：

1. 学术检索
2. 三步分析
3. 项目知识沉淀
4. 写作闭环

## 3. 首页定位差异

### 原文档问题

首页与工作台、聊天、流程入口边界不清，容易理解成“所有功能聚合门户”。

### v2 收敛

首页只做：

1. 搜索
2. 搜索结果恢复
3. 本次搜索的三步分析

## 4. 优先级差异

### 原文档问题

很多能力被并列陈述为核心功能，缺少现实优先级。

### v2 收敛

P0 优先处理：

1. 搜索可用性
2. 搜索缓存恢复
3. 项目知识工作台
4. 写作状态一致性
5. 基础删除与兼容能力

## 5. 技术假设差异

### 原文档问题

默认假设以下能力会自然存在：

1. 数据源都能稳定接入
2. 数据库结构与代码同步
3. 全文检索引擎、LangGraph、协作体系可顺滑落地

### v2 收敛

明确写出：

1. 数据源存在限流/授权/反爬约束
2. 旧库需要 schema compatibility
3. 前端历史缓存结构需要兼容迁移

## 6. 对原文档的处理建议

### README

建议保留为：

1. 项目介绍
2. 愿景说明
3. 技术背景

但不再作为当前版本实施依据。

### 原 PRD

建议保留为：

1. 愿景 PRD
2. 长期能力蓝图

但需要在文首或目录处明确标注：

> 当前实施请以 `docs/产品/PRD_v2_产品收敛版.md` 为准。

## 7. 当前最重要结论

不是原文档错误，而是项目已经进入“产品收敛阶段”，需要一个更贴近实际代码与用户反馈的实施型 PRD。
