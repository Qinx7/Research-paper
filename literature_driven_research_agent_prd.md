# 文献驱动科研助手需求文档

## 1. 产品名称

**文献驱动科研助手**

英文名：

**Literature-driven Graduate Research Agent**

## 2. 当前产品定位

本产品面向研究生科研场景，围绕“文献检索 → 选题分析 → 项目设计 → 知识沉淀 → 论文写作 → 通用 PPT 展示”构建稳定闭环。

当前版本不再提供：

- 开题报告生成
- 开题报告下载
- 基于开题报告的 PPT 生成链路

## 3. 核心目标

### 3.1 解决的问题

1. 用户不知道如何从模糊研究想法快速进入可执行方向
2. 用户不知道如何从大量文献中识别研究热点与研究空白
3. 用户缺少把文献、成果、证据和资料沉淀为项目知识库的能力
4. 用户在论文写作时缺少可追溯依据和连续编辑工作台
5. 用户希望快速生成可展示的通用 PPT 或 HTML Deck

### 3.2 当前主链路

```text
输入研究问题
→ 学术检索
→ 需求理解
→ 文献分析
→ 研究方向生成
→ 项目设计
→ 成果/资料入知识库
→ 论文草稿生成与编辑
→ 通用 PPT / HTML Deck 生成
```

## 4. 目标用户

### 4.1 主要用户

1. 准备选题或准备开题阶段的研究生
2. 已进入项目实施阶段、需要沉淀知识与材料的研究生
3. 正在论文写作阶段、需要依据追踪与生成辅助的研究生

### 4.2 次要用户

1. 导师
2. 课题组成员

当前版本不以团队协作和导师评审为核心交付目标。

## 5. 核心功能需求

### 5.1 学术检索

- 支持多来源学术检索
- 支持中英文范围切换
- 支持检索来源状态展示
- 支持检索历史恢复与删除

### 5.2 文献分析与研究方向

- 自动完成需求理解
- 自动总结研究热点、研究趋势与研究空白
- 生成多个可研究方向
- 对研究方向进行评分
- 支持研究方向承接为项目当前方向

### 5.3 项目设计

- 基于当前研究方向生成结构化项目设计
- 项目设计可保存、可恢复、可继续用于写作与展示

### 5.4 项目知识库

- 上传成果文件
- 解析 PDF / 文本资料
- 文档切分与知识入库
- 文献笔记、证据卡片、资料片段统一沉淀
- 支持项目内资料搜索

### 5.5 论文写作

- 创建草稿
- 生成章节
- 连续编辑全文
- 查看章节依据
- 合规检查
- DOCX / PDF 导出

### 5.6 通用 PPT / HTML Deck

- 根据项目设计生成通用 PPT
- 根据论文草稿生成 HTML Deck
- 根据手动大纲生成 HTML Deck
- 支持多风格主题选择

## 6. 明确删除的旧能力

以下能力属于已移除范围，不应再作为实现目标或对外承诺：

1. 开题报告 Agent
2. `proposals` 表
3. `/api/proposal/*` 接口
4. `/api/ppt/proposal` 接口
5. 基于 `proposal_id` 的 HTML Deck 预览

## 7. 当前版本成功标准

1. 首页搜索后，可稳定恢复历史记录，且不会重复生成已有分析结果
2. 研究方向与项目设计可保存并继续承接
3. 上传资料后，可进入项目知识库并在写作中被引用
4. 写作页可稳定完成章节生成、编辑、依据查看和导出
5. 通用 PPT / HTML Deck 能力可从项目设计或论文草稿正常生成

## 8. 当前版本接口边界

### 保留接口

- `POST /api/literature/search`
- `POST /api/literature/analyze`
- `POST /api/research/directions`
- `POST /api/research/design`
- `POST /api/drafts/...`
- `GET /api/ppt/styles`
- `POST /api/ppt/generate`
- `POST /api/ppt/html-deck`

### 移除接口

- `POST /api/proposal/generate`
- `GET /api/proposal/{proposal_id}`
- `GET /api/proposal/project/{project_id}/latest`
- `GET /api/proposal/{proposal_id}/download`
- `POST /api/ppt/proposal`

## 9. 技术约束

1. 先保证主链路稳定，不再扩展已下线的开题报告能力
2. 数据库结构通过 Alembic 迁移维护
3. 产品优先面向 PC 端
4. 文档与实现必须保持一致，不再宣称已删除能力
