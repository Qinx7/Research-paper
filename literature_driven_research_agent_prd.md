# 文献驱动型研究生科研 Agent 需求文档

## 1. 项目名称

**文献驱动型研究生科研设计与成果生成 Agent**

英文名称：

**Literature-driven Graduate Research Agent**

---

## 2. 项目定位

本项目旨在构建一个面向研究生科研全过程的智能 Agent 系统。

系统能够根据用户提供的研究需求，自动检索和分析相关文献，发现研究热点与研究空白，提出可研究方向，并进一步完成研究课题设计、开题报告生成、开题 PPT 生成。

在用户后续完成系统开发、实验、数据分析或项目成果后，系统还可以辅助撰写论文、制作毕业答辩 PPT 和整理科研成果。

本系统不是单纯的论文写作工具，而是一个以：

- 文献检索
- 文献分析
- 研究方向发现
- 项目设计
- 开题材料生成
- 论文与 PPT 生成

为核心的研究生科研全过程智能体。

---

## 3. 核心目标

系统的核心目标是帮助研究生完成从“模糊研究想法”到“可开题课题”，再到“论文和答辩成果”的完整科研流程。

核心流程如下：

```text
用户输入研究需求
↓
自动检索相关文献
↓
分析研究现状、热点与不足
↓
提出多个可研究方向
↓
评估方向的创新性与可行性
↓
用户选择研究方向
↓
进行项目设计
↓
生成开题报告
↓
生成开题 PPT
↓
用户后续上传项目成果或实验数据
↓
生成论文
↓
生成毕业答辩 PPT
```

---

## 4. 系统核心价值

本系统重点解决研究生科研过程中的以下问题：

1. 不知道自己的研究兴趣可以转化为什么课题；
2. 不知道如何查找和筛选相关文献；
3. 不知道如何从文献中发现研究空白；
4. 不知道如何提出可开题、可实施、可写论文的研究方向；
5. 不知道如何设计完整研究方案；
6. 不知道如何撰写开题报告；
7. 不知道如何制作开题 PPT；
8. 后续有项目成果后，不知道如何转化为论文和答辩 PPT。

## 5. 目标用户

### 5.1 准备开题的研究生

## 8.1 研究需求理解 Agent

### 功能说明

系统需要理解用户输入的模糊研究需求，并提取其中的关键信息。

### 输入示例

```text
我想研究人工智能在高校教学中的应用，最好能结合大语言模型，做出一个系统，后续可以写毕业论文。
```

### 系统应识别

```text
研究领域：人工智能教育应用
核心技术：大语言模型、RAG、教育智能体
应用场景：高校教学、课程问答、学习支持
可能成果：系统原型、实验数据、论文、PPT
适合方法：文献综述、系统设计、实验评价、问卷调查
```

### 输出内容

- 研究领域识别；
- 核心关键词提取；
- 可能研究对象；
- 可能研究方法；
- 适合成果形式；
- 初步研究建议。

---

## 8.2 文献检索 Agent

### 功能说明

根据用户需求自动生成检索关键词和检索式，并从多个学术数据库查询相关文献。

### 支持文献来源

优先支持开放或可通过 API 访问的数据源：

| 数据源 | 用途 |
|---|---|
| OpenAlex | 综合文献检索 |
| Semantic Scholar | 论文摘要、引用、相关论文 |
| Crossref | DOI、期刊、出版信息 |
| arXiv | 计算机、数学、物理等预印本文献 |
| PubMed | 医学、生物方向 |
| IEEE Xplore API | 工程、计算机、电气方向 |
| ACM Digital Library | 计算机方向 |
| Zotero API | 用户个人文献库同步 |

### 检索能力

- 中文关键词生成；
- 英文关键词生成；
- 同义词扩展；
- 布尔检索式生成；
- 按年份筛选；
- 按引用量筛选；
- 按期刊或会议筛选；
- 按开放获取筛选；
- 多数据源去重。

### 示例关键词

用户需求：

```text
大语言模型在高校教学中的应用
```

系统生成中文关键词：

```text
大语言模型、生成式人工智能、ChatGPT、智能教学、教育智能体、课程问答、个性化学习、RAG、学习分析
```

系统生成英文关键词：

```text
large language model
LLM
generative AI
ChatGPT
AI in education
intelligent tutoring system
educational chatbot
retrieval augmented generation
personalized learning
learning analytics
```

### 示例检索式

```text
("large language model" OR "LLM" OR "ChatGPT" OR "generative AI")
AND
("higher education" OR "university teaching" OR "educational chatbot" OR "intelligent tutoring")
AND
("retrieval augmented generation" OR "RAG" OR "personalized learning")
```

## 8.3 文献筛选与质量评估 Agent

### 功能说明

对检索到的文献进行筛选和排序，推荐最适合作为研究基础和开题依据的文献。

### 筛选维度

| 维度 | 说明 |
|---|---|
| 主题相关性 | 是否与用户需求高度相关 |
| 发表年份 | 是否为近五年文献 |
| 期刊/会议质量 | 是否发表在高质量来源 |
| 引用量 | 是否具有较高学术影响 |
| 方法完整性 | 是否包含明确研究方法 |
| 数据与实验 | 是否有真实实验或数据支撑 |
| 系统设计价值 | 是否可为用户项目设计提供参考 |
| 研究空白价值 | 是否暴露了可继续研究的问题 |

### 输出示例

```text
推荐优先阅读以下 10 篇核心文献：

1. Paper A
推荐理由：
- 直接研究大语言模型在高校教学中的应用；
- 提出了教育问答系统框架；
- 可为后续系统设计提供参考。

2. Paper B
推荐理由：
- 研究 RAG 在教育问答中的效果；
- 提供了实验评价方法；
- 适合作为后续实验对比的 baseline。
```

## 8.5 研究热点与研究空白发现 Agent

### 功能说明

根据多篇文献的分析结果，总结当前领域的研究热点、发展趋势和研究空白。

### 输出内容

#### 研究热点

```text
当前研究热点包括：

1. 基于 ChatGPT 的教学辅助；
2. 基于 RAG 的课程知识库问答；
3. 个性化学习反馈生成；
4. 教育智能体系统设计；
5. 学习者行为分析与智能推荐；
6. 生成式 AI 的教育伦理与可信性问题。
```

#### 研究不足

```text
现有研究主要存在以下不足：

1. 很多研究停留在理论探讨，缺少真实教学场景验证；
2. 普通大语言模型容易产生知识幻觉；
3. 很多教育问答系统缺少课程知识库支撑；
4. 缺少对学生学习效果的量化评估；
5. 缺少教师和学生双主体使用体验分析；
6. 对系统可解释性和可信度关注不足。
```

#### 可研究切入点

```text
1. 基于 RAG 的高校课程智能问答系统设计与应用研究；
2. 面向个性化学习反馈的大语言模型教学 Agent 研究；
3. 生成式人工智能辅助高校课程教学的效果评估研究；
4. 基于课程知识图谱与大语言模型融合的教学问答平台设计；
5. 面向学生自主学习的教育智能体系统设计与评价。
```

## 8.6 可研究方向生成 Agent

### 功能说明

根据文献分析结果，生成多个可开题、可实施、可写论文的研究方向。

### 每个研究方向应包含

```text
1. 研究题目
2. 研究背景
3. 研究问题
4. 研究目标
5. 研究内容
6. 研究方法
7. 数据来源
8. 预期成果
9. 创新点
10. 可行性分析
11. 风险分析
12. 适合论文类型
```

### 示例研究方向

#### 方向一：基于 RAG 的高校课程智能问答系统设计与应用研究

```text
研究背景：
随着大语言模型的发展，智能问答系统在高校教学中的应用逐渐增多。
但普通大语言模型容易产生知识幻觉，且难以准确回答课程相关问题。

研究问题：
如何利用 RAG 技术提升高校课程问答系统的准确性和可信度？

研究目标：
设计并实现一个基于课程资料知识库的 RAG 智能问答系统，并通过实验评估其回答准确率、相关性和用户满意度。

研究内容：
1. 构建课程知识库；
2. 设计 RAG 检索与生成流程；
3. 实现高校课程智能问答系统；
4. 构建测试问题集；
5. 对比普通 LLM 与 RAG 系统的回答效果；
6. 进行用户满意度调查。

研究方法：
文献研究法、系统设计法、实验对比法、问卷调查法。

数据来源：
1. 课程教材；
2. 教学 PPT；
3. 教学大纲；
4. 测试问题集；
5. 学生使用评价数据。

预期成果：
1. 一个课程智能问答原型系统；
2. 一套课程知识库；
3. 实验对比数据；
4. 用户满意度分析；
5. 毕业论文；
6. 开题和答辩 PPT。

创新点：
1. 将 RAG 技术应用于具体高校课程问答场景；
2. 从准确性、可信度和满意度多维度评价系统效果；
3. 提供可复用的课程知识库构建流程。

可行性：
较高。系统开发难度适中，数据可由课程资料构建，适合研究生毕业论文。
```

## 8.7 研究方向评估与推荐 Agent

|---|---|
| 文献基础 | 是否有足够文献支撑 |

| 论文写作价值 | 是否容易形成完整论文结构 |
| 开题通过率 | 是否适合作为开题题目 |
| 成果展示性 | 是否适合 PPT 展示和答辩 |

### 评分表示例

# 16. 学术合规要求

## 16.1 文献必须真实

所有参考文献必须来自真实数据库，不能由大模型编造。

要求：

```text
1. 每条参考文献必须有 DOI、URL 或数据库来源；
2. 文献元数据需要保存；
3. 引用内容需要与原文观点匹配；
4. 生成参考文献前需要进行校验。
```

---

## 16.2 研究方向必须可行

系统不能只生成看起来高大上的题目，而要考虑：

```text
1. 研究生时间周期；
2. 用户技术能力；
3. 数据是否容易获取；
4. 实验条件是否具备；
5. 论文结构是否完整；
6. 开题通过可能性。
```

---

## 16.3 开题阶段不能伪造成果

开题阶段可以写：

```text
1. 拟开展研究；
2. 计划完成实验；
3. 预期成果；
4. 预计创新点；
5. 研究方案设计。
```

不能写：

```text
1. 实验结果表明；
2. 已经证明；
3. 系统显著提升；
4. 研究已取得明显效果。
```

---

## 16.4 论文阶段必须基于真实成果

毕业论文和答辩 PPT 中的以下内容必须来自真实数据或真实项目成果：

```text
1. 实验结果；
2. 数据分析；
3. 用户评价；
4. 模型性能；
5. 系统截图；
6. 结果图表。
```

## 17.1 风险：AI 编造文献

解决方案：

```text
1. 所有文献必须来自真实 API；
2. 文献保存 DOI、URL、来源数据库；
3. 生成参考文献前进行校验；
4. 不允许模型自由生成参考文献。
```

---

## 17.2 风险：研究方向不切实际

解决方案：

```text
1. 对每个方向进行可行性评分；
2. 明确数据来源和实施条件；
3. 对技术难度进行分级；
4. 提供低风险替代方案。
```

---

## 17.3 风险：开题报告内容空泛

解决方案：

```text
1. 开题内容必须绑定文献依据；
2. 研究内容必须细化到任务级别；
3. 技术路线必须可执行；
4. 预期成果必须具体。
```

---

## 17.4 风险：论文阶段生成虚假结果

解决方案：

```text
1. 论文结果部分必须基于用户上传数据；
2. 没有实验结果时只能生成实验设计；
3. 所有图表和数据需要保存来源；
4. 系统明确区分“预期结果”和“实际结果”。
```

# 20. 一句话总结

本项目要构建的是一个文献驱动型研究生科研 Agent。

它能够根据用户提供的研究需求自动查找相关文献，分析研究现状和研究空白，提出可研究方向，完成项目设计，并生成开题报告和开题 PPT。

在用户后续完成项目成果后，系统还可以根据真实成果辅助撰写论文和生成毕业答辩 PPT。

最核心的第一版应该优先打通：

```text
需求输入 → 文献检索 → 文献分析 → 研究方向推荐 → 项目设计 → 开题 PPT
```

后续再扩展到：

```text
项目成果 → 论文撰写 → 毕业答辩 PPT
```
